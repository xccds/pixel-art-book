# 给程序员的像素艺术课

> 程序员学美术最大的误区，不是不会画，而是不会看。
>
> 观察能力先于绘画能力。这也是本书与你读过的大多数美术教程最大的区别。

---

也许你经历过这样的场景。

花了半年写完一个游戏，玩法打磨了上千次。但画面里到处都是默认方块、拼凑的素材，或者 AI 生成但风格完全不统一的图片。代码已经完成了 80%，却始终不能称为一个真正的游戏。

如果你也是这样，这本书就是写给你的。

## 这本书是什么

一本**写给程序员**的像素艺术教材。它不要求你有绘画基础。它要求的东西更可控——每天投入一点时间，用对的方法练对的东西。

这本书只做一件事：**用你熟悉的思维方式，教你从零到做出可运行的游戏视觉。** 每一个概念都从你熟悉的东西出发——像学编程一样学美术：先理解语法（观察），再写代码（练手），最后搭建完整项目（制作）。你不是在学一门全新的技能，你是在一个你已经熟悉的思维框架里，填充新知识。

## 这本书不是

- ❌ 不是教你怎么画插画——这本书教的是**游戏美术**（可组合、可动画、可引擎导入的素材），不是独立的图画
- ❌ 不是"像素艺术简史"或"像素艺术家访谈"——每章有练习、有合格标准、有上手行动
- ❌ 不是工具说明书——本书教的是原理和决策

## 学完之后，你将获得什么

- 能独立制作一套 16×16 或 32×32 的**角色、地块和 UI 素材**，可直接导入游戏引擎
- 能分析一张优秀像素作品，并**说出它为什么好、为什么不好**——不是"好看/不好看"，而是"明度结构清晰、负空间呼吸感足够"
- 能建立适合自己项目的**视觉风格文档**，像 `.editorconfig` 一样约束所有素材的一致性
- 能把 Aseprite、AI 反馈、Godot 等工具串联成**一条完整的生产管线**
- 更重要的是——你会拥有一套可以**持续提升的学习方法**，而不是依赖灵感或天赋

## 这本书怎么读

你不必从头读到尾。观察01 有一份自评量表，做完它你就知道该从哪里开始。三能力（看/做/整）分层诊断，缺哪层补哪层。

但无论你的起点在哪，有一条铁律不要违反：**先练看，再练画。** 你的眼睛必须先学会识别问题，手才能去修问题。

## 这本书的承诺

**这本书用的是你的语言。** 每一个概念都从你熟悉的东西出发——不是让你变成艺术家，是让你用程序员的思维方式，系统性地学会"看"和"做"。

**这本书不要求你会画。** 99% 的程序员拿着本书之前做不出一张能用的游戏素材——这是正常的起点。错的不是你，是那些告诉你"先画一个圆，然后细化"的教程——它们是教插画家的，不是教你的。
