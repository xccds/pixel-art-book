# 像素艺术完全指南 —— 一个程序员的看与画

> **给那个打开过 Krita、对着空白画布发了半小时呆的你** —— 你不是没有美术天赋，你只是从来没有被用对的方法教过。

---

## 这本书是什么

这是一本**写给程序员**的像素艺术教材。它不要求你有绘画基础，不假设你有"天赋"。它要求的东西更具体——每天投入一点时间，用对的方法练对的东西。

这本书只做一件事：**用你熟悉的思维方式（系统、分层、约束、迭代），教你从零到做出可运行的游戏视觉。**

它不是一本教你"怎么画得好看"的书。它是一本教你**怎么系统地学会看、学会做、学会整**的书——这和学一门编程语言是同一种过程。

## 这本书不是

- ❌ 不是一本教你怎么画插画的教材——这本书教你的是**游戏美术**（可组合、可动画、可引擎导入的素材），不是独立的图画
- ❌ 不是一本"像素艺术简史"或"像素艺术家访谈"——每章有练习、有合格标准、有上手行动
- ❌ 不是一本工具的说明书——工具操作收在制作01 和附录G 里，正文教的是原理和决策

## 这本书怎么读

你不必从头读到尾。观察01 有一份自评量表，做完它你就知道该从哪里开始。三能力（看/做/整）分层诊断，缺哪层补哪层。

但无论你的起点在哪，有一条铁律不要违反：**先练看，再练画。** 程序员最容易犯的错误是一上来就打开软件画——因为你习惯了打开 IDE 写代码。美术不是这样——你的眼睛必须先学会识别问题，手才能去修问题。

## 这本书的承诺

**这本书不要求你有天赋。** 它要求的东西更可控：每天投入一定的时间，用对的方法练对的东西。

**这本书不要求你会画。** 99% 的程序员拿着本书之前做不出一张能用的游戏素材——这是正常的起点，不是你的错。错的不是你，是那些告诉你"先画一个圆，然后细化"的教程——它们是教插画家的，不是教你的。

**这本书用的是你的语言。** 每一个概念都从你熟悉的东西出发——函数、数据结构、约束系统、协议栈、编译管线。你不是在学一门全新的技能——你是在一个你已经熟悉的思维框架里，填充新知识。
