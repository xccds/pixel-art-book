## 二、核心设计决策（已与用户确认）

1. **形态**：统一为一本（非主书+配套、非重新架构）
2. **用途**：个人学习用，版权无虑，书 B 内容与提取图均可直接用——但因下条统一为 SVG
3. **结构修正**：书 A 的五子能力坍缩为**三能力（看 / 做 / 整）**，每个对应结构里的部；"选兵器"因工具固定坍缩进"制作"开头
4. **文风**：书 B 全部内容**全面改写**为书 A 的程序员第一人称 + 编程类比声调
5. **图像**：**全书插图统一原创 SVG**，不用书 B 提取位图；书 A 100 张 SVG 全面审计修复；书 B 插图点按内容需要增减（非 1:1 复现，可合并/拆分/增删，~113 为基线参考）
6. **部名定稿**：观察 · 练手 · 风格 · 制作 · 继续（双字、平行、平实）

**三能力映射**：
- 看（能力1：感知+词汇）→ 第一部 观察
- 做（能力2：执行+工具）→ 第二部 练手（训练）+ 第四部 制作（应用，工具固定=Aseprite+Godot）
- 整（能力3：整合）→ 第四部 制作后半（制作07 上引擎 / 制作08 生产管线 / 制作09 一致性审计）
- 风格（方向决策）与 继续（维持+扩展）为衔接与延展，不对应单层能力

## 六、文风改写规范

1. 第一人称"你"，把读者当程序员同行
2. **每个新概念配一个编程类比**（数据结构/设计模式/工具链/协议/算法）——书 B 融入内容必须补类比
3. 保留书 A 结构装置：
   - 每章"这一章解决什么问题"开头
   - "程序员类比" callout
   - "如果只记住一句话"
   - "上手行动"
   - 章末"小结 + 扩展阅读 + 引注"
4. 书 B 的对比表格保留（表格中性，两书通用）
5. 书 B 的 step-by-step 保留，但在步骤前后加类比框
6. 术语中英并行（明度/Value，抖动/Dithering…）
