# 练手01 线条：从颤抖到控制

### 1.0 这一章解决什么问题

第一部「观察」结束了。你现在有了一套视觉词汇（观察02 的八概念）、一套分析法（观察03 的 VTS）、一个喂眼睛的习惯（观察04）、一个 24 小时反馈回路（观察05）。但"会看"不等于"会做"——你能说出一条线为什么歪，不等于你能画直它。第二部「练手」就是把"看"转成"做"：从最基础的结构单元开始，逐个攻克八概念的像素技法。第一个就是**线条**。

观察02 里线条是八种"数据类型"之一，那里只给你一句话定义和两个游戏极端（《Shovel Knight》硬黑轮廓 vs《Hyper Light Drifter》隐形轮廓）。这一章把那一个概念彻底拆开：线在像素网格上到底是怎么"画"出来的、为什么会歪、怎么修、以及——它在一个角色身上有哪几种"勾边策略"可选。

更重要的是：**本章 L1 起开始一个角色项目——画它的轮廓/线稿。** 这个角色不会画完就丢：它将在练手05（色彩）被上色，在制作02（像素角色工作流）做成完整可导入引擎的角色。线条是它的骨架，骨架歪了，后面所有上色和动画都是把歪斜放大。所以这一章不只是练线，是在给你的第一个角色打地基。

这一章不是教你"把线画直"，是教你"理解线在游戏画面里做什么，以及怎么用程序员的脑子控制它"。你只需要 Aseprite（或任何支持 1px 铅笔的像素编辑工具），加一支笔一张纸。

### 1.1 程序员视角：线条到底是什么

#### 1.1.1 像素线不是"画"出来的，是"算"出来的

如果你用纸笔画线，你的手自然地做了一条连续曲线。但在像素画布上，根本没有"连续"这回事。你面对的是一张二维网格，每个格子要么填要么不填。所谓的"线"，本质上是在这张网格上一串 1 和 0 构成的**路径选择问题**。

换个角度看：你在 CSS Grid 里定义一个元素要从左上角走到右下角，你会怎么做？`grid-column: 1 / span 1; grid-row: 1 / span 1`，然后 `grid-column: 2; grid-row: 2`，一步步走出一根对角线。每一步你都做一个决策：这一格往右走几步，往下走几步？**像素画线和你用 grid 布局在逻辑上完全一样——你只是在操作一个离散坐标系里的路径。**

这个路径有两个参数：`dx`（水平偏移）和 `dy`（垂直偏移）。它们的比率 `dx/dy` 就是**斜率（Slope）**。在整数网格里，斜率只能取有限几种"合法值"：

- **1:1（dx = dy）**：每步右移 1、下移 1。45° 对角线，像棋盘上"象"的走法。
- **2:1（dx = 2, dy = 1）**：每步右移 2、下移 1。约 26.6° 的浅角对角线。
- **1:0（dx 任意，dy = 0）**：没有垂直分量。纯水平或纯垂直——像素画中最"干净"的线。

这三种是最基础、最安全的入门斜率。为什么是这三种？因为在整数坐标系里，一条像素线要看起来"平滑"，相邻像素必须连成一条不产生突兀断裂的路径。1:1 已是最陡的非垂直斜率（你没法用 0.5 个像素做偏移），1:0 是最平坦的非零偏移。这中间的任何角度——3:1、4:1、5:2——理论上可以存在，但它们要么看起来像"出错的对角线"，要么需要配合特别的曲线上下文才不跳眼（图 练手01.1）。

![像素线条三种基本斜率](images/练手/练手01-fig01-像素斜率.svg)

*图 练手01.1：像素线条的三种基本斜率规则。在离散网格中，"画线"就是一串 (dx, dy) 偏移指令。*

> **程序员类比：** 像素斜率 = CSS Grid 的列宽比例。`grid-template-columns: 1fr 1fr` 就是 1:1（两列等宽）；`2fr 1fr` 就是 2:1（第一列是第二列两倍）。Bresenham 直线算法干的就是这件事——在离散网格上用整数运算画一条视觉最平滑的线。读懂 Bresenham 的决策逻辑（每一步在两个候选像素里选误差更小的那个），你就明白了为什么 2:1 在像素中最"自然"。

#### 1.1.2 线条在游戏画面中的四种功能

你不需要每条线都画得像工程师图纸。线条在游戏美术里有四种不同用途，每种对"线质"的要求完全不同：

**第一种：界定轮廓（Contour）。** 一根实线把前景和背景切开——角色外轮廓、UI 按钮边框、对话框分割。核心要求是**清晰**：哪怕只给玩家 0.3 秒扫一眼，他也得知道"边界在哪"。2D 游戏里统一 1px 或 2px 的轮廓通常就够了；游戏越暗黑压抑（如《Limbo》），轮廓越粗越黑；游戏越明亮轻快（如《星露谷物语》），轮廓越细或干脆用色块代替。

**第二种：引导视线（Leading Line）。** 画面中一根若有若无的曲线，玩家不会"看见"它，但眼睛会不由自主跟着走。蜿蜒的山路、倾斜的剑、飘动的头发——这些不是装饰，是画面里的**视觉 hyperlink**：把视线从一个东西锚定到另一个。网页设计里你用 whitespace 和 alignment 控制阅读顺序；游戏画面里你用引导线控制玩家先看哪。

**第三种：表达动势（Gesture）。** 线条的方向和张力能传达运动状态。从左上挥向右下的粗线暗示"砍击"；从下往上渐细的曲线暗示"跳跃"；层层扩散的弧线暗示"冲击波"。动势线可以完全不符合现实，但玩家大脑会自动翻译成动作。关键是**加速度感**——线宽变化和弯曲程度要对齐你想要的运动节奏。

**第四种：暗示质感（Texture Line）。** 密集短线阵列能模拟木纹、石缝、发丝。这类线的重点不是每一根画得完美，而是**密度控制**——多少根聚一起、间距多少、方向是否一致。密度均匀 = 光滑质感（金属、皮肤）；密度随机 = 粗糙质感（树皮、地面）。

四种功能不互斥。一根武器轮廓线可以同时做轮廓（定义形状）和视线引导（枪口指向敌人），顺便暗示质感（刀锋金属反光）。你分析一张图时不需要把四种全找出来，但要知道"这根线在这里承担什么任务"——这直接决定你该用 1px 还是 2px、直线还是曲线、实线还是虚线。

### 1.2 理论最小集

不讲艺术史，只讲像素画里立刻用得上的几条规则。

#### 1.2.1 斜率法则 = 阶梯原理（同一件事）

像素美术社区有个老说法叫**阶梯原理（Staircase Principle）**：同一段线条里，每一"级台阶"的像素数应保持一致——线条越陡，每级台阶像素越多；不要在一串 2 像素台阶里突然夹一个 1 像素台阶。这跟上面 1.1.1 的"路径规划/斜率规则"是**同一件事的两种说法**：阶梯原理是社区从结果倒推的直觉总结，斜率法则是从路径规划的数学结构正推的精确描述。两者合起来记一句话：

> 一条像素斜线要想看起来平滑，每一步的斜率必须保持在 1:1、2:1 或 1:0 之一；斜率一旦跳变——比如从 2:1 突然变 4:1——人眼就在那个突变点感知到"断了一截"。

这跟数字信号处理里的采样定理是同一个逻辑：你的斜率序列就是一个离散信号，相邻采样值变化太剧烈就产生"高频噪声"——在这里就是锯齿感。保持斜率变化量不超过 1，相当于对信号做了一次低通滤波。

由此推出三条具体规则：

- **直线**：水平/垂直走 1:0，最干净；对角线走 1:1，最平滑。能用这两者就别用别的。
- **斜线（对角线的浅角版本）**：走 2:1。当必须用特定角度时，交替使用两种相邻线段长度（比如 2 像素段和 3 像素段），别跳到差 2 以上的组合。
- **曲线**：见下一节，台阶宽度要渐变，不能跳变。

#### 1.2.2 Jaggies（锯齿感）的成因与修复

Jaggies 是像素美术最常见的问题。它不是你画工不好导致的——是**算法选择**导致的。

想象你在画一条从左到右逐渐弯曲的线，每一行决定这一段的水平像素数。初学者通常凭"眼睛感觉"放像素，结果水平段像素数跳来跳去：5、2、6、3……这个序列的每次跳变，在人眼里就是一个"锯齿"（图 练手01.2）。

![锯齿感错误 vs 正确斜率修正](images/练手/练手01-fig02-锯齿对比.svg)

*图 练手01.2：左——水平段长度跳变剧烈，视觉感知为锯齿边缘。右——段长每次只变 0 或 1，感知为平滑曲线。*

**Jaggies 产生的精确条件：** 相邻两行水平像素段长度差 ≥ 2，即 `|len[i] - len[i+1]| >= 2`。社区里还有一条等价的直觉版规则：**不要让一排像素被更大的像素包围**——一排 1px 的细段夹在两排 5px 的粗段中间，那 1px 就是锯齿的尖角。

**修复方法不是重画，是"雕琢"。** 像雕刻家一样，从粗线条里削去多余的像素：先用铅笔工具画一条粗略的线，放大观察找出不符合阶梯原理的地方，用橡皮或铅笔削去/补上 1-2 个像素，让段长回归渐变。通常改 2-3 行就够。

#### 1.2.3 曲线平滑技巧：台阶宽度渐变序列

这是实操中最重要的技巧。来自像素美术社区的长期实践（ashes999 的像素画教程有详细论述）[1]：画一条从水平渐变到接近垂直的平滑曲线时，每一行的水平像素数应形成这样一个序列：

```
[5, 4, 3, 2, 2, 1, 1, 1, 2, 3]
```

变化规律 5→4→3→2→2→1→1→1→2→3，几个关键特征：

- 每次变化不超过 1，没有任何一步跳 2 或更多。
- 中间平缓区（三个连续的 1）形成曲线"顶点"——曲率最大的区域。
- 序列对称：5→3 递减和 1→3 递增形成对称曲线。

![曲线平滑技巧——台阶宽度渐变序列](images/练手/练手01-fig03-曲线平滑序列.svg)

*图 练手01.3：台阶宽度渐变序列。规律——相邻段的像素数变化不超过 1，曲线就平滑。*

> **程序员类比（linter）：** 把每一行的水平像素数存进数组 `step_widths[]`，写一个检查函数确保 `Math.abs(step_widths[i] - step_widths[i+1]) <= 1` 对所有 `i` 成立。这条规则就是你的"linter"——不符合它的线段，重新画。你不需要每次都真去跑这个函数，但你要养成"画完曲线默念一遍差值"的习惯，就像写完代码默念一遍边界条件。

**操作步骤：**

1. 画完曲线后，从最上面一行开始，逐行数出每段水平像素数。
2. 写下来，检查相邻差。
3. 如果某个差值 ≥ 2，回到那一行，加或减 1-2 个像素，让差值回归 ≤ 1。
4. 再看一遍整体效果——通常改 2-3 行就够。

画曲线时还有两条配套规则：**避免孤立像素（lone wolf）**——单独凸出的一个像素会破坏平滑感；**善用复制翻转**——对称的曲线部分只需画一半，复制翻转即可，既省工又保证对称。

#### 1.2.4 线宽与情绪

线条粗细不是装饰选择——它直接改变玩家对画面的情绪感知。规律根植于人类视觉对物理世界的经验：粗重的东西给人力量感和威胁感（粗树干比细枝更"稳"），细轻的东西给人精致感和脆弱感（蛛丝、发丝）。

| 线宽 | 视觉感受 | 适合场景 |
|------|---------|---------|
| 1px | 精致、锐利、干净 | 像素角色轮廓、UI 图标、细字体 |
| 2px | 稳重、清晰、有重量 | 像素角色外轮廓（粗一圈制造剪影感）、按钮边框 |
| 3-4px | 力量感、粗糙、沉重 | 大型敌人轮廓、爆炸效果、标题文字 |
| 不均匀 | 有机感、手绘感、动态 | 草稿风格、能量特效、毛发纹理 |

**核心原则是一致性。** 大多数像素艺术里，线条宽度默认 1px——最清晰、最受推荐。当你选了 2px 外轮廓，所有外轮廓都得是 2px。风格选择可以自由，但必须保持一致。在同一画面里，线宽切换必须有"理由"：如果角色右手武器用 1px 但左手盾牌用 3px，玩家大脑会尝试解释——"盾更重？"或"画错了？"。你不打算传达信息，就不要制造线宽差异。一种常见的高级用法是**粗外轮廓 + 细内部细节**，营造纸雕感；某些角色或场景用更粗的线传达特定质感。但这些都是"有理由的破例"，不是"随便切"。

关于线条的情绪表达能力，图 练手01.4 给了最直接的演示——你不需要画整幅画，三根线就够选择合适的"情绪词汇"。

![线条情绪——五组不同排列的线条表达五种情绪](images/练手/练手01-fig04-线条情绪.svg)

*图 练手01.4：线条情绪对比。方向、曲率和间距共同决定传达的情绪——平静、紧张、流动、爆发、蜷缩。*

#### 1.2.5 五种轮廓类型——勾边策略

线四功能里第一种"界定轮廓"是最常用的，而"怎么勾边"在像素艺术里有一套成熟的策略菜单。不同轮廓能彻底改变一个角色的视觉效果。这一节把五种主流轮廓类型拆开，并给你一个程序员视角的类比来记它们。

**1. 无轮廓（No Outline）。** 不用轮廓线，靠色块本身区分形状。常见于背景元素。即使没线，仍需消除锯齿——色块边缘就是你的"线"。

**2. 黑色内描（Black Inline）。** 轮廓和内部细节都用黑色线条。8 位时代的产物，受限于硬件色数。缺点是让精灵显得"脏"——每一处内部边界都被黑线强调，信息密度过高。

**3. 黑色外轮廓（Black Contour）。** 只有外部边缘用黑色，内部用彩色。最常见的风格，能让精灵从背景中脱颖而出，同时保持内部干净。这是大多数游戏精灵的默认选择。

**4. 彩色轮廓（Colored Outline）。** 轮廓用与内部颜色对应的深色版本——每个部分的轮廓颜色都不同。比黑色外轮廓更精致、更融合，色彩更丰富干净。

**5. 着色轮廓（Shaded Outline）。** 轮廓线本身带光影变化，根据光源方向变深浅。像素艺术中最复杂的轮廓类型，在背景里融合得最好，立体感最强。

| 轮廓类型 | 适用场景 | 风格特点 |
|---------|---------|---------|
| 无轮廓 | 背景、无轮廓风格 | 柔和、自然 |
| 黑色内描 | 复古 8 位风格 | 怀旧、粗犷 |
| 黑色外轮廓 | 大多数游戏精灵 | 清晰、突出 |
| 彩色轮廓 | 精致插画 | 色彩丰富、干净 |
| 着色轮廓 | 大型精灵、背景 | 立体感强、融合性好 |

![五种轮廓类型对比](images/练手/练手01-fig05-五种轮廓对比.svg)

*图 练手01.5：同一个简单角色在五种勾边策略下的视觉差异。轮廓不是"加不加线"的问题，是"边界检查严到什么程度"的问题。*

> **程序员类比：** 轮廓 = 类型系统的**边界检查策略**。五种轮廓对应五种类型严格度：
> - **无轮廓 = `any` / 不检查**：快、灵活，但形状容易和背景粘在一起（运行时才在边界出错）。
> - **黑色内描 = 到处显式标注**：像每个变量都写 `Array<number>` 的遗留代码，安全但啰嗦，视觉信息过载（"脏"）。
> - **黑色外轮廓 = 严格外边界 + 内部推断**：API 表面严格、内部靠上下文推断——默认的 strict 模式，清晰且不啰嗦。
> - **彩色轮廓 = 泛型 / 参数化类型**：每个边界按各自域类型着色，更精确、更表达领域语义。
> - **着色轮廓 = refinement types / 依赖类型**：边界本身携带运行时变化的信息（光与影），最强大也最复杂，与上下文融合最深。
>
> 选哪种不是"哪个最好"，是"你的项目需要多严格的边界、能负担多复杂的类型系统"。背景 tile 用 `any`（无轮廓）省事；主角用 strict（黑色外轮廓）保证可读；大 BOSS 用 refinement（着色轮廓）追求融合感。但一个项目里**风格必须统一**——不能一个文件 strict 一个文件 `any`，那等于没类型系统。

#### 1.2.6 边缘处理

"边缘"指的是形状与背景交界的地方。好的边缘处理能极大提升质感，规则只有三条：

- **避免锯齿边缘**：轮廓线每一级台阶都遵循 1.2.1 的阶梯原理。
- **适当使用双线**：某些区域用 2px 宽的线强调体积感（呼应 1.2.4 的"粗外轮廓"）。
- **外轮廓对比度**：确保轮廓与背景有足够对比度，否则"边界检查形同虚设"。

**内部即外部。** 物体内部的细节边界——亮面与暗面的交界线——同样需要平滑，不能有锯齿。阶梯原理不只管外轮廓，也管一切"色块与色块的邻接处"。这条会在练手04（明度）和练手06（质感）里反复用到。

形状组合（用基础形状拼出复杂物体）留到练手02 深讲，这里只提一句：大多数令人难忘的游戏角色都是基础形状的组合——吃豆人 = 切掉一块的圆，马里奥蘑菇 = 圆柱 + 半圆 + 矩形。你画的第一个角色也该这么开始。

### 1.3 练习

这三个练习按难度递进，每个都配了具体合格标准。你不需要一次做完——建议分三天，每天一个。

> **贯穿式角色项目（重要）：** 本章 L1 起开始一个角色项目——画它的轮廓/线稿。这个角色将在**练手05（色彩）**上色，在**制作02（像素角色工作流）**做成完整可导入引擎的角色。所以 L1 你画的不是随便一根线，是你未来角色的骨架。选一个你愿意跟它相处好几章的角色——一个骑士、一个史莱姆、一个机器人、一个魔法师都行，32×32 或 32×64。

#### L1 · 角色剪影与线稿（约 30-60 分钟）

**目标：** 同时练"线条情绪"直觉和"轮廓界定"能力，并给你的贯穿角色起一个骨架。

**你需要：**

- Aseprite，新建 32×32 或 32×64 画布
- 一张纸和一支不能擦的笔（签字笔/圆珠笔）

**步骤：**

1. 先在纸上用 3-4 根线条快速勾出你角色的**剪影**（silhouette）——单一颜色的实体形状，不画内部细节。限时 5 分钟，画 3 个不同姿势的变体。
2. 退一步看：哪个剪影即使全黑也能认出"这是个骑士/史莱姆/机器人"？挑最好的一个。剪影可识别是优秀角色设计的第一步——想想马里奥、吃豆人、索尼克，你仅凭轮廓就能认出他们。
3. 在 Aseprite 里，用 1px 铅笔把选中的剪影落到画布上。先只画**外轮廓**，不填色。
4. 选一种轮廓策略（1.2.5）：先试黑色外轮廓（最稳妥），如果你想要更柔和可以试无轮廓或彩色轮廓。记下你选了哪种、为什么。
5. 给轮廓内部加 2-3 处关键细节线（眼睛位置、武器轮廓、一道衣纹），用 1px。
6. 全程用 1.2.3 的"台阶宽度渐变"规则检查曲线部位，用 1.2.1 的阶梯原理检查斜线部位——发现锯齿就雕琢修复。
7. 保存。这个文件就是贯穿项目的起点，命名建议 `character-v01-outline.aseprite`，之后每章迭代都开新版本号。

**合格标准：**

- [ ] 纸上 3 个剪影中至少有 1 个，给一个不认识你的人看，他能猜对角色类型（"是个骑士"或"是个机器人"）。
- [ ] Aseprite 里的外轮廓没有 |相邻段差| ≥ 2 的锯齿——逐行数过水平像素数确认。
- [ ] 轮廓策略选择有写下一句理由（"选黑色外轮廓因为我想让角色从背景跳出来"）。
- [ ] 文件按 `character-v01-outline` 命名保存，准备好被练手05 继续上色。

**关键提示：** 不要想着"画得好不好看"。这一步测的是**轮廓能不能界定身份** + **线条有没有遵守斜率规则**。歪歪扭扭但剪影可辨、曲线无锯齿，就是过关。

---

#### L2 · 无锯齿曲线（30-60 分钟）

**目标：** 掌握像素画中消除 jaggies 的核心技巧，独立于角色项目练一次纯技术。

**你需要：**

- Aseprite，新建 64×64 画布

**步骤：**

1. 用 1px 铅笔画一条从左到右、先降再升的 S 形曲线。不要用曲线工具——手动画，一个像素一个像素点。
2. 画完后从最上有像素的那行开始，逐行数出每行水平像素数，记下来，例如 `[3, 5, 4, 2, 6, 1, 5, 3]`。
3. 检查相邻差值，把差值 ≥ 2 的地方标记出来。
4. 修改那些行——加或减像素——直到所有相邻差值 ≤ 1。
5. 反复步骤 2-4 直到序列满足"渐变"条件。
6. 保存最终结果。

**合格标准：**

- [ ] 序列中任意相邻差值 ≤ 1（`[5,4,3,2,2,1,1,1,2,3]` 通过；`[5,1,5,3,1,3,2]` 不通过）。
- [ ] 缩小到 100% 比例（无缩放），肉眼看不到明显锯齿断裂点。
- [ ] 无孤立像素（lone wolf）凸出。

**提示：** 考的不是"手稳"，是"眼细"。前两次你可能完全看不出 jaggies 在哪——正常。你需要训练眼睛识别 1-2 个像素的"不对劲"。坚持数序列，比纯靠眼可靠。

---

#### L3 · 视线引导线分析（1-3 小时）

**目标：** 学会在一张完成的游戏截图里"读出"设计师埋下的视线引导线——把 1.1.2 的第二种功能用到分析上。

**你需要：**

- 一张你觉得"看着舒服"的游戏截图
- 一个能画线的标注工具（Aseprite 新建图层手动画线，或 Figma / Photoshop 画笔）
- 纸和笔

**步骤：**

1. 把截图导入标注工具。
2. 不做任何分析，先盯 30 秒。回答：**"眼睛最先看哪里？然后看哪里？最后看哪里？"** 写下来。
3. 新建图层，用**红色线条**画至少 3 条引导你视线的线（已有边缘、道具指向、光影分界、角色视线方向——任何牵引眼球的视觉元素）。
4. 每条线起点标"入"、终点标"出"。
5. 回答并写下来：
   - 这 3 条线终点分别指向哪个元素？
   - 哪一条对你的眼球牵引力最强？为什么？（对比度？方向？长度？）
   - 去掉其中一条，画面视觉流会怎么变？
6. 找一张你觉得"看着不舒服"的截图（构图混乱、不知看哪），重复 1-5。对比两次分析，写 100-200 字的发现。

**合格标准：**

- [ ] 每条引导线有明确起点和终点标注。
- [ ] "舒服"截图分析中，至少一条引导线终点指向画面焦点（主角脸、关键道具、CTA 按钮）。
- [ ] "不舒服"截图分析中，能说出至少一个"为什么混乱"的原因（如"引导线全指向边缘没一条指中心"或"两条引导线交叉导致视线来回跳"）。
- [ ] 两段对比文字至少含一个自己总结的结论（用自己话，不引用本章）。

> **程序员类比：** 这个过程像分析代码库的**控制流（Control Flow）**——你追踪从 `main()` 开始会依次调用哪些函数。"不舒服"的截图相当于一段没注释、没命名规范、goto 满天飞的代码——视线不知往哪跳，跳一次丢一次上下文。

### 1.4 常见错误与诊断

初学者在像素画线条里最常踩的三个坑。

**错误 1：斜率不一致 → 锯齿感。** 曲线整体还行，拉近到 100% 时边缘像被虫咬过——某处突然凸/凹 1-2 像素。人眼对轮廓上的"不连续点"极敏感（进化上轮廓凹凸可能意味着捕食者或食物）。**原因：** 曲线中间某处斜率突变，比如正走 2:1（每行 2 像素），突然某行放了 4 个（跳到 4:1），下一行又回 2。**修复：** 找到那段，逐行数水平像素数，把跳变行修正（周围是 2 你放了 4，改 3 或保持 2），遵循 1.2.3 的台阶渐变规则。

**错误 2：线宽随意切换 → 风格断裂。** 同一 sprite 里头部轮廓 1px、手臂 2px、刀刃 1px 但刀柄 1px+黑填——这些差异没传达任何信息，像不同人拼贴。**原因：** 你在不同时间、不同缩放、不同心态下分别画了不同部位，没有全局线宽规范。**修复：** 先定整张 sprite 的线宽规则（外轮廓 X px / 内部细节 Y px / 最细装饰 1px），画完全局检查一遍只看线宽。Aseprite 技巧：按住 Shift 保持铅笔工具，用 `[` `]` 快速切换笔刷大小，防手滑。

**错误 3：忘记视线引导 → 画面散乱。** 所有东西都"在画面上"，但玩家扫一眼不知该往哪看。**原因：** 你把精力全放在"画得对"（形状、色彩、比例），没分配注意力给"眼睛走位"。"画得对"和"看得顺"是两套判断体系——前者是客观规范，后者是视觉心理学。**修复：** 画完退两步眯眼看（眯眼降色彩辨识但保留明度和方向感），拿红笔在画面上画出"视线路径"，从第一眼落到的地方开始。如果路径没指向意图焦点，加一根引导线、调一个对比度、或移开干扰元素，重复直到红线终点落在意图焦点上。

### 1.5 本章小结

线条不是用手画出来的——在像素世界里，它是用斜率规则**算**出来的。你不需要"手稳"，但你需要理解 1:1、2:1、1:0 三种基本斜率，以及相邻水平段长度变化不超过 1 这条核心规则（社区的"阶梯原理"和"路径规划/斜率法则"是它的两种说法）。这两点加起来，就是你消除锯齿感的全部工具箱。

线本身也是画面的**结构**——界定轮廓、引导视线、表达动势、暗示质感。这四种功能是每次下笔前该在脑中过一遍的 checklist。轮廓这一项还有五种"勾边策略"可选——无、黑内描、黑外轮廓、彩色、着色——选哪种等于选你角色的"类型系统严格度"。

最后，线条控制力只能靠**反复犯错再修正**获得。L1 练直觉和轮廓（还起了贯穿角色的骨架），L2 练技术，L3 练分析。三个练习做完，你至少不会画出一条自己都不知道为什么歪的线——而且你有了一个能在后面几章继续长大的角色。

**如果只记住一句话：** 像素线是路径规划问题，不是手绘问题——先让斜率合规，再谈美观。

### 1.6 扩展阅读

1. **《Pixel Logic》— Michafrar（michafrar.com）**  
   像素美术领域最系统的入门书。第 3 章"Lines and Curves"是对本章斜率规则的深度扩展，含更多斜率变化模式和复杂曲线实战。这本书本身就是用像素画排版和插图，值得一看 [2]。

2. **Bresenham 直线算法（Wikipedia 搜 "Bresenham's line algorithm"）**  
   计算机图形学经典算法，解决的核心问题和你画的每一条像素线完全一样：在离散网格中用整数运算画一条视觉最平滑的线。读懂 Bresenham，你就明白为什么 2:1 在像素中最"自然"——重点是它的决策逻辑：每一步在两个候选像素里选误差更小的那个 [3]。

3. **Derek Yu 的像素画教程（derekyu.com/makegames/pixelart.html）**  
   《Spelunky》创作者分享的像素画方法论。他特别强调"限制"（Constraint）在像素画中的价值——线宽统一、色板限制、格子大小限制——这些限制不是束缚，而是帮你做决定的框架。和本章的"线宽一致性"思想一致。

4. **Aseprite 官方教程——线条与曲线（aseprite.org/tutorial）**  
   打开 Aseprite，Help → Tutorials，有专门的线条和曲线练习。跟着画一遍，第二手经验变成身体记忆。

5. **《The Animator's Survival Kit》— Richard Williams，第 1-2 章**  
   虽然讲动画，但前两章对线条"动势"和"节奏"的讨论可直接迁移到游戏美术。Williams 用极简线条示范如何通过粗细变化和方向暗示传达重量、速度、情绪——是 1.2.4 的理论根源之一 [4]。
