# 练手02 形状与体积：剪影说了算

### 2.0 这一章解决什么问题

练手01 你给贯穿角色画了轮廓/线稿——一个能在黑色填充后仍被认出"是骑士/史莱姆/机器人"的剪影骨架。但骨架只是 Shape（2D 轮廓），它还是张纸片。这一章解决两件事：**怎么让那张纸片长出体积（Form），以及怎么确保它缩到游戏里的实际尺寸后还认得出来。**

观察02 把"形状与体积"列为八种数据类型之一，给了你一句话定义和两个极端（《Dead Cells》高速动作中保剪影可读，《Minit》1-bit 极简靠几何辨识）。这一章把它彻底拆开：剪影为什么是第一道单元测试、圆/方/三角各自暗示什么、五几何体怎么拼出万物、金属和布料为什么不能用同一套色板、以及——小尺寸下"可读性"为什么压倒一切细节。

这一章是全书最复杂的一次合并：书 A 的剪影测试和形状心理学打底，书 B 的五几何体光照和材质区分接上"体积"那一半，书 B 的可读性与象征画法接上"小尺寸"那一半。三块拼成一条流水线：**剪影（认得出）→ 体积（有质感）→ 可读（缩得住）**。

你只需要 Aseprite，加一支笔一张纸。

### 2.1 程序员视角：剪影是视觉设计的单元测试

#### 2.1.1 形状 = 数据结构的 Schema 声明

如果你写过 TypeScript，这个类比应该能让你会心一笑：

```typescript
// 形状（Shape）= interface / type 声明
// 体积（Form）= 实例化的 object
interface CharacterSilhouette {
  headShape: "round" | "elongated" | "rectangular";
  bodyProportion: number;   // 头身比
  asymmetry: boolean;       // 左右不对称更有辨识度
  negativeSpace: number;    // 负空间占比（胳膊和身体之间的空隙）
  distinctProps: string[];  // 独特道具：["sword", "cape", "horns"]
}

// 剪影测试 = 单元测试：只测最小可验证单元
// 通过了 → 放心加细节；失败了 → 重写 interface，别急着实例化
function silhouetteTest(c: CharacterSilhouette): boolean {
  const fill = "#000000";   // 填黑
  const size = 32;          // 缩小到游戏内尺寸
  return humanVerification(fill, size);  // 必须返回 true
}
```

你写代码时不会先写 500 行逻辑再一口气跑测试——你会先写一个函数、跑一个测试、通过后继续。视觉设计完全一样：**函数体** = 角色内部细节（纹理、颜色、装备）；**函数签名** = 角色剪影（轮廓、比例、姿态）；**单元测试** = 填黑 + 缩小 + 能否辨认。函数签名不对，实现再漂亮调用方也用不上。同理剪影不过关，加再多装饰也救不了——噪杂的游戏画面里，剪影是玩家唯一能稳定接收的信号。

另一个角度：**剪影测试 ≈ API 合约设计**。API 在被调用之前，你得先想好输入输出。剪影就是角色和玩家眼睛之间的 API 合约——玩家看一眼，就知道这对象是友方还是敌方、可交互还是背景。合约不清晰，玩家就误触、误判、摔键盘。

总结一句：**在暗色背景中只靠 fill:black 就能辨认——这是所有视觉设计的第一道单元测试。**《守望先锋》的角色设计师在概念阶段每个英雄都必须过严格的剪影测试 [1]，这不是偶然，是纪律。

#### 2.1.2 剪影测试工作流

这是几乎所有 3A 工作室角色设计师的日常工作流 [1][4]：

1. **生成纯黑剪影**：把角色填成纯黑（#000000），放在中灰或浅色背景上。
2. **缩小到游戏内尺寸**：角色在游戏中占 64×64，就把剪影缩到 64×64。
3. **放同级对象中对比**：主角、敌人、NPC、可交互物体的剪影同框，检查能否区分。
4. **有任意一个混在一起**：回去改 Shape，不通过不进上色阶段。
5. **极端测试**：喷枪模拟运动模糊、缩到 32×32 甚至 16×16，再测一次。

![剪影测试：填黑→缩小→还能辨认吗？](images/练手/练手02-fig02-剪影测试.svg)

*图 练手02.2：左侧——清晰人形 + 道具 + 动势，缩到 32px 仍能识别；右侧——模糊块状，缩小后变成一坨。*

剪影辨识度三个要素：**不对称性**（完美对称让剪影平庸，加披风/单肩甲/举起的手立刻有辨识度）；**负空间**（胳膊和身体之间的空隙和"实的部分"一样重要，好的剪影是实-空-实-空的节奏）；**独特道具轮廓**（武器/帽子/背包要"跳出来"，猎空双枪、马里奥帽子、林克盾牌都是高辨识度锚点）。

#### 2.1.3 从剪影到三视图：四步法

很多程序员（包括我学画前几年）的错误是：上来就画带颜色、带阴影、带细节的"完成品"，不好看就改，几小时过去还是一坨。正确流程四步，**每步不可跳**：

![从剪影到三视图：角色成型四步法](images/练手/练手02-fig03-剪影到三视图.svg)

*图 练手02.3：剪影（辨识度）→ 结构（比例）→ 细节（造型）→ 色彩（氛围）。顺序不能乱。*

**Step 1 纯黑剪影**——只关心轮廓，不画比例细节颜色。唯一问题：缩小后能认出来吗？画 5 个挑最好的。不过关别进下一步。**Step 2 内部结构线**——画关键分割线：头在哪、身体分几段、四肢长度比例。用中轴线（定对称）、腰带线（定上下半身）、肩线（定宽度）。**Step 3 细节雕刻**——在结构框架内加面部、服装、道具、纹理线，所有细节必须服务剪影而非破坏它；某条装饰线填掉了关键负空间——删掉。**Step 4 色彩与光影**——最后一步。前三步对了，上色是"给建好的房子刷漆"；前三步错了，上色是"给垃圾喷金漆"。

这套流程是"防御性设计"：**每步都有明确通过标准，不通过不前进**——跟你写代码给每个模块写测试再集成的逻辑完全一样。

### 2.2 形状心理学：圆/方/三角 + affordance

人类大脑对几何形状有天生的情感反应——不是文化习得，而是常见倾向，不是不可违背的定律 [2]。

| 形状 | 心理属性 | 游戏案例 | Affordance（功能暗示） |
|------|---------|---------|----------------------|
| **圆形** | 友好、柔软、安全、萌 | 卡比、《星之卡比》 | "可交互，不会伤害你" |
| **方形** | 可靠、稳重、笨重、可预测 | Minecraft Steve、推箱子 | "稳定平台 / 可以推动" |
| **三角形** | 危险、尖锐、攻击、不稳定 | 塞尔达敌人、尖刺 | "别靠近 / 碰到会受伤" |

![形状心理学：三种基本形状的语言](images/练手/练手02-fig01-形状心理学.svg)

*图 练手02.1：圆=安全、方=稳定、三角=危险。玩家在看到颜色和细节之前，形状已经在传递信息。*

**圆形**——自然界中圆的东西通常软（果实、蛋、婴儿脸），尖锐的东西通常危险（牙齿、荆棘）。圆形角色 = "我是你朋友"。UI 里圆角按钮暗示"安全默认操作"——跟你给 CTA 加 `border-radius: 50%` 同理。**方形**——传递"稳定""可靠"。Minecraft 一切皆方块，物理规则一目了然：能站、能堆、能推。方形角色暗示"可预测，不会突然做奇怪行为"。**三角形**——暗示危险、运动、威胁。塞尔达几乎所有敌人轮廓都含尖锐三角；平台跳跃里一块尖朝上的地板不用任何文字，玩家就不踩——这就是 affordance。

**Affordance**（功能暗示）是设计师 James Gibson 提出的概念 [3]：一个正方体告诉玩家"可以推"，圆形按钮告诉"可以按"，三角形陷阱告诉"绕开走"。好的形状设计不需要文字说明——玩家直觉替你完成所有教学。

> **程序员类比：** 基本形状 ≈ 基本数据类型（int/float/string），组合后像 struct 定义新类型。圆角按钮 ≈ "安全的默认操作"，红色警告图标 ≈ "危险操作"——形状自身就在做类型标注（type annotation），玩家大脑在 0.1 秒内完成类型检查。

### 2.3 从形状到体积：数字雕塑与五几何体

#### 2.3.1 Shape 在先，Form 在后

厘清两个概念：**Shape（形状/轮廓）** 是对象在平面上的二维边界，把角色压扁成纸片——外轮廓就是 Shape，所有剪影测试测的都是 Shape。**Form（体积/体量）** 是对象在空间中的三维存在感，一个球和一张圆形的纸 Shape 可能相同，Form 完全不同——Form 告诉大脑"这东西有厚度，能占空间"。

游戏里两者缺一不可：Shape 决定"能不能认出来"，Form 决定"有没有质感"。纯平面角色像贴纸；Shape 模糊但 Form 强的角色像一坨真实但不知所云的泥。**Shape 在先，Form 在后**——先确保轮廓过关，再加体积感，顺序不能倒。

像素画看似二维（在平面上放方块），但好的像素画本质上是对三维世界的模拟。一个球的像素画不是"圆形的二维形状"，而是"一个被光照亮的三维球体"在网格上的投影。这种**形体思维**（Form Thinking）是传统美术训练的核心——不把物体看作线条和色块的平面组合，而是三维形体在二维平面上的投影。低分辨率下每个像素都承担着暗示三维体积的任务。

#### 2.3.2 体积感的三种制造方法

有了好 Shape，下一步给它 Form。三种独立方法：

![体积感的三种制造方法](images/练手/练手02-fig04-体积感三方法.svg)

*图 练手02.4：光影法（明暗）/ 透视法（结构线）/ 叠压法（前后遮挡）。三者可组合。*

**方法一：光影法（Chiaroscuro）**——模拟光源照射，朝光面亮、背光面暗、挡光处投影。最强大的体积制造法，掌握光影一个二维圆就能看起来像三维球，但学习曲线最陡。独立游戏开发者建议先学"单光源 + 硬阴影"，效果/成本比最高。**方法二：透视法（Perspective / Construction Lines）**——用消失点和辅助线"证明"物体有空间深度。画一个正方形是平的，画三个可见面（顶/前/侧）就是立体的。这是 Drawabox 课程 [5] 训练的核心：几百个盒子练下来，手和眼学会"任何形状在透视下该长什么样"。透视法不需要理解光影，纯靠线条就能制造体积。**方法三：叠压法（Overlap）**——最简单也最易被忽略：A 遮住 B，A 就在 B 前面。不需要光影不需要透视。2D 游戏尤其有用：树遮房子一角、房子遮角色脚——三层叠压就在纯二维画面上创造清晰空间深度。

**三种可组合**：光影 + 透视 = 写实 3D；叠压 + 光影 = 有质感的扁平插画；三者都强 = 专业水平。

#### 2.3.3 五大基本几何体：万物皆可拆解

掌握了三种体积方法，下一步是把"任何物体"拆成可处理的基本单元。所有复杂物体都可以**拆解**为五种基本几何体的组合——掌握这五种的光影规律，就等于掌握了万物。

![五大基本几何体的光照模式](images/练手/练手02-fig05-五几何体光照.svg)

*图 练手02.5：球/方/柱/锥/环，光源统一来自左上方。每个形体的亮面、明暗交界、暗面、反射光位置不同。*

- **球体**：无平面，法线连续变化。高光是靠光源侧的小点（光滑）或模糊区（粗糙）；明暗交界线是弧形；暗面底部有**反射光**（地面反弹回来照亮暗部）。像素里常用 3~5 级明度弧形排列模拟曲面。
- **立方体**：只有面，没有渐变。顶面最亮、侧面中间调、正面最暗，三面**阶跃式**切换，边缘用清晰色块切换不要渐变。
- **圆柱体**：顶/底是平面（明度均匀），侧面像球体的水平渐变——但高光是一条**垂直亮线**而非点。像素里做 3~4 个垂直条带，从左到右渐变。
- **圆锥体**：可理解为"渐变宽度的圆柱"——顶窄底宽，高光从顶到底逐渐增宽，明暗既有垂直方向也有水平方向。
- **环体**：最复杂，有**自遮挡**——环的部分身体挡住光线投阴影到其他部分；顶部外侧受光最多，内侧底部被自身遮挡处于阴影，内侧上半可能被环境光照亮。

一旦掌握基本几何体，你就能用"几何体分析法"看一切物体：

| 物体 | 几何体拆解 |
|------|-----------|
| 人体头部 | 球体（头骨）+ 半圆柱（颈部） |
| 人体躯干 | 圆柱/椭圆柱（胸腔）+ 椭圆柱（腰部） |
| 树木 | 圆柱（树干）+ 球体/锥体（树冠） |
| 剑 | 细长方体（剑身）+ 长方体（剑柄）+ 球体（剑柄头） |
| 城堡 | 多个立方体 + 圆柱（塔楼）+ 锥体（屋顶） |
| 角色 | 球（头）+ 圆柱/椭圆柱（四肢）+ 椭圆柱（躯干） |

> **程序员类比：** 五几何体 = 图形渲染的基本图元（primitives），像 WebGL 的 box/sphere/cylinder 几何体——复杂 mesh 都是它们的组合。物体拆解表 = struct 的字段声明：`struct Character { head: Sphere; torso: Ellipsoid; limbs: Cylinder[4]; }`。会拆 = 会画，因为每个字段的光影规律你都已掌握。这就是"复制形状让你完成一幅画；理解形体让你能画出任何角度"。

### 2.4 材质区分：换皮不换骨

掌握基本形体的光影后，下一步是让同一形体换上不同材质。**不同材质对光的反应不同**——同一个球，金属和布料的色板完全不一样。

![材质区分：同一形体换五种皮肤](images/练手/练手02-fig06-材质对比.svg)

*图 练手02.6：同一个球体，光源不变，只改"着色器参数"。金属高对比锐利高光，布料柔和几乎无高光，皮肤暗面偏红，玻璃透明锐利白高光，木材漫反射无高光。*

- **金属**：高对比度，亮暗跳跃极大；锐利高光（1~2 像素亮点）；暗面明显反射环境色。色板用明度跳跃大的阶梯。
- **布料**：柔和过渡，亮暗差距小，几乎无高光；阴影分布跟随褶皱走向。色板用相邻明度差小的阶梯，用条纹暗示褶皱。
- **皮肤**：**次表面散射**（Subsurface Scattering）——光穿透皮肤表层再反射出来，导致暗面微微发亮、偏红。明暗比金属柔和比布料略锐利，高光模糊。暗面偏向紫红而非纯黑是表现"肉感"的关键。
- **水/玻璃**：高透明度能看到后面物体；高光极亮且锐利（白色）；边缘用非直线暗示折射。不要画水的"颜色"，画水对下方和周围物体颜色的影响。
- **木材**：低对比度漫反射，几乎没有锐利高光；纹理沿纤维方向延伸。基色上用略深/略浅的线条暗示木纹。

| 材质 | 对比度 | 高光特征 | 暗面特征 | 色板建议 |
|------|--------|---------|---------|---------|
| 金属 | 高 | 锐利、小面积 | 反射环境色 | 明度跳跃大的阶梯 |
| 布料 | 低 | 几乎无高光 | 柔和、偏冷 | 相邻明度差小的阶梯 |
| 皮肤 | 中 | 柔和区域高光 | 偏暖（偏红） | 带色相偏移的阶梯 |
| 玻璃/水 | 极高 | 锐利、白色 | 透射底色 | 高对比但少量颜色 |
| 木材 | 低 | 无高光 | 柔和 | 相邻明度差小的阶梯 |

> **程序员类比：** 材质 = 着色器（shader）的参数集。同一网格，换一套 `specular`（高光强度/锐利度）/ `diffuse`（漫反射）/ `subsurface`（次表面散射）/ `transmission`（透射）参数 = 换一种材质。光影结构（几何体的亮暗分布）不变，只是渲染管线换了配置。金属 = 高 specular 低 diffuse；布料 = 高 diffuse 低 specular；皮肤 = 开 subsurface；玻璃 = 开 transmission + 极高 specular。你不动 mesh，只动 material。

### 2.5 AO 与反射光：让精灵"扎根"

两个进阶光影概念，是把像素画从"还行"推向"专业"的秘诀。

**环境光遮蔽（Ambient Occlusion，AO）**——物体表面相互靠近的角落处光线无法进入，形成更暗阴影。常见应用：手臂与身体之间的缝隙、下巴与脖子的夹角、物体与地面接触处、叶片根部——都加深 1~2 级。实现方法：在原本阴影色之上再加深 1~2 级用于这些"接触角落"。一个没有 AO 的精灵像漂浮在平面上，加上 AO 立刻有"扎根感"。

**反射光（Bounced Light / Reflected Light）**——光打到地面或其他表面后反弹到物体暗面。出现在暗面靠近地面的位置，亮度低于直接光照但明显比纯阴影亮，颜色受反弹面影响（绿色地面→暗面带绿）。像素里在暗面和轮廓之间插 1~2 像素宽的稍亮颜色，通常偏向反弹面色相。

> **程序员类比：** AO = 环境光的"接触阴影缓存"——那些光线照不到的缝隙，相当于代码里"边界条件的死角"，单独算一遍加暗。反射光 = 全局光照（GI / indirect lighting）的手算版本——你手动模拟光在场景里弹一次。两者都是把"直接光照模型"扩展成"更接近真实"的近似，代价只是多 1~2 个像素的色阶。

形体思维实战：看到一个参考图，别复制它的二维形状，问自己——这物体由哪些基本几何体组成？光源从哪来？每个几何体的亮/中间调/暗/阴影在哪？什么材质？哪部分该加 AO？暗面要不要反射光？六问完，你就能画出任何角度。

### 2.6 可读性：小尺寸下的第一优先级

前面讲的都是"画得对"，这一节讲"缩得住"。**可读性（Readability）** 指观众能多快、多准确地理解你的像素画"画的是什么"。在小尺寸下，可读性是最重要的设计考量——比细节、纹理甚至色彩更重要。一个 16×16 的精灵里，角色的表情、姿势、身份都要用最少的像素表达。

精灵越小传达信息越难。尺寸决定你能展示什么：

| 精灵尺寸 | 设计重点 | 注意事项 |
|---------|---------|---------|
| **小（<32×32）** | 可识别的特征 | 省略细节，追求剪影清晰 |
| **中（32×64）** | 平衡细节与清晰度 | 确保关键特征突出 |
| **大（>64×64）** | 丰富的细节与动画 | 保持线条干净，避免过度复杂 |

小像素画里移动一个像素就可能改变观众对整幅画的解读。经典案例：《宝可梦 红/蓝》里一个 16×16 的"Game Boy 男孩"，本意是男孩在玩 Game Boy，很多玩家看成男孩举着杯子——只差几个像素。教训：**小尺寸下简洁通常是最好的选择**，别过度加细节，坚持简单形状。

![可读性检查：100% 实尺 + 模糊测试](images/练手/练手02-fig07-可读性检查.svg)

*图 练手02.7：左——100% 实尺可读，剪影 + 关键锚点（剑/羽/腰带扣）活下来；中——模糊后焦点仍在；右——堆了 8 色装饰，剪影变碎，不知看哪。*

**可读性检查四法**：1. **缩略图预览**——始终在 100% 缩放（无放大）下检查作品。2. **模糊检查**——用模糊滤镜看精灵，模糊后看起来奇怪就有可读性问题；活下来的特征才是真焦点，被抹掉的是可删的装饰。3. **他人测试**——让别人看，别告诉是什么，看能否准确辨认。4. **"少即是多"**——有疑虑，删减而不是增加。

> **程序员类比：** 可读性 = API 的**最小惊讶原则**（Principle of Least Astonishment）。小尺寸下信息密度要克制——接口暴露的字段越少，调用方越不容易误用。模糊测试 ≈ 压测（stress test）/降级测试：扛住降级还活下来的字段才是真契约，被模糊抹掉的就是"可以下个版本再加"的可选参数。100% 实尺检查 = 在目标环境（production）跑一遍，别只在 8 倍放大下自我感觉良好。

### 2.7 象征画法与比例：用符号代替写实

小尺寸下你没法写实，得学会用"符号"代表复杂物体。

**手的画法**——手是最难在小尺寸下表现的部分。核心：先用块面画出手套形状，再加细节。从形状开始不从线条开始（线条在小尺寸太难）；先画"手套"再加手指；只用 3 根手指 + 拇指（卡通常见手法）；用颜色区分不同手指。**眼睛的画法**——眼睛是角色精灵的视觉焦点，人类天生被脸吸引，眼睛质量决定表现力。一个高光像素就能改变眼睛表情；眼睛通常定义你的风格。**面部表情**——极小尺寸（如 16×16）可能没空间画眼睛，这时可通过脸部阴影暗示眼睛存在。

> **程序员类比：** 象征画法 = 用 icon/emoji 代替写实渲染。就像 UI 里用 🔧 图标代替"设置"二字、用 ⚠ 代替警告文本——符号比写实信息密度更低但传达更快。3 根手指的手 = 一个高度抽象的"手 icon"，牺牲解剖准确度换可读性，跟组件库里把复杂控件简化成图标的逻辑一样。

**角色比例**——头身比极大影响风格和表现力：**大头比例（Q 版/chibi）** 头大，更多空间表现面部表情，可爱风格；**写实比例** 头小，更多空间表现身体语言和动作。同一角色在不同场景可用不同比例：世界地图用 Q 版，战斗中可能更写实。

**剪影设计**——好的角色设计该有清晰剪影，去掉所有内部细节仅凭轮廓也能辨认。这一条和 2.1 的剪影测试是同一件事的两面：2.1 教你怎么测，这里提醒你——可读性的根基还是剪影。颜色区分角色（主角红、反派蓝）只在彩色模式有效，黑暗场景、色盲模式、黑白画面下全失效；剪影在任何条件下稳定。**不要移除颜色**——颜色是重要辅助信号，原则是颜色是增强不是替代：好形状 + 好颜色 = 优秀；差形状 + 好颜色 = 还是认不出来。

### 2.8 练习

三个练习按难度递进，建议分三天。**L1 接练手01 的贯穿角色**——给你在练手01 画的轮廓骨架长出体积。

> **贯穿式角色项目（续）：** 练手01 L1 你画了 `character-v01-outline.aseprite` 的剪影/线稿。本章 L1 在它上面**长体积**——用五几何体思维给那个轮廓上光影。这个角色还将在**练手04（明度）**深化明暗结构、**练手05（色彩）**正式上色、**制作02（像素角色工作流）**做成可导入引擎的完整角色。所以 L1 你画的不只是练习，是给你贯穿角色的第一层"肉"。

#### L1 · 给贯穿角色长体积（约 45-90 分钟）

**目标**：把练手01 的平面剪影变成有体积感的角色，练"几何体拆解 + 光影法"。

**你需要**：练手01 的 `character-v01-outline.aseprite`、Aseprite、纸笔。

**步骤**：
1. 打开练手01 的线稿。先在纸上把你的角色**拆成几何体**：头=球、躯干=椭圆柱、四肢=圆柱、武器=方体……画一个小拆解草图。
2. 定单一光源方向（建议左上方），在 Aseprite 新建图层，给每个几何体部位置亮面/中间调/暗面。先用 3 级明度，不碰颜色。
3. 明暗交界线用弧形（球/柱）或阶跃（方体），别用渐变工具——手点。
4. 在暗面底部加 1 像素反射光，在手臂与身体缝隙、下巴与脖子夹角加 AO（再深 1 级）。
5. 全程保持练手01 的外轮廓不变——这一步只加内部光影，不动 Shape。
6. 缩到 100% 看：角色是不是从"纸片"变成了"有厚度的东西"？保存为 `character-v02-volume.aseprite`。

**合格标准**：
- [ ] 纸上有几何体拆解草图，每个部位标了对应几何体类型。
- [ ] 单一光源方向一致，所有部件亮暗方向不矛盾。
- [ ] 角色至少有 3 级明度（亮/中间/暗），球体部位明暗交界是弧形非直线。
- [ ] 至少 2 处 AO（接触角落）+ 1 处反射光。
- [ ] 外轮廓与练手01 一致，剪影测试仍通过（填黑缩小仍可辨）。
- [ ] 文件存为 `character-v02-volume`，准备交给练手04。

#### L2 · 材质换皮（约 30-60 分钟）

**目标**：理解"同一形体不同材质"——光影结构不变，只换色板参数。

**步骤**：
1. 在 Aseprite 画一个球体（约 24×24），光源左上，3~5 级明度。
2. 复制三份，分别用**金属、布料、皮肤**的材质特征着色——不改变光源方向和几何体的亮暗分布，只改色板阶梯的选择（金属用大跳跃 + 锐利 1px 高光；布料用小跳跃无高光；皮肤暗面偏红）。
3. 三份并排，检查"同一个球三种气质"。

**合格标准**：
- [ ] 三份的亮暗位置完全一致（几何体光影没变）。
- [ ] 金属有 1~2px 锐利高光 + 暗面带环境色；布料无锐利高光 + 过渡柔和；皮肤暗面偏红。
- [ ] 不看标签，别人能指出哪个是金属/布料/皮肤。

#### L3 · Drawabox 250 盒子挑战（前 50 个，约 1-3 小时，可选）

**目标**：训练空间透视和体积感——所有绘画的基础能力。

**步骤**：
1. 访问 [Drawabox Lesson 1](https://drawabox.com/lesson/1) 学基础，进 [250 Box Challenge](https://drawabox.com/lesson/250boxes)。
2. A4 纸 + 0.5mm 黑色针管笔。每天 10 个盒子，坚持 5 天。每个盒子：纸面上方画水平地平线；在地平线上/下方画 Y 字形（三分支=盒子三可见边缘）；用钢笔（不用尺子）从 Y 延展完整盒子画三可见面；最后用红笔画"被遮挡的边缘"验证透视。
3. **必须用钢笔不能擦**——逼你在下笔前先在脑子里想好每一笔，这是练空间想象力不是练书法。

**合格标准**：
- [ ] 50 个盒子里至少 40 个"三条可见边"收敛方向正确（朝同一个大方向收拢）。
- [ ] 红笔检查时隐藏边缘交叉点大致对齐。
- [ ] 不追求完美——50 个之后你能"感觉到"一个盒子在空间中怎么旋转。

**为什么是盒子不是球？** 球体没有"面"看不出透视。盒子的三个面是透视训练最佳对象——面方向、边长比例、收敛角度，每个维度都在锻炼空间判断力。Drawabox 创始人 Irshad Karim 说：**250 个盒子做完，你眼中的世界会从"线条"变成"空间"**[5]。

### 2.9 常见错误与诊断

**错误 1：剪影辨识度不够——"缩小后糊成一团"。** 角色 100% 缩放时不错，缩到游戏实际尺寸后主角敌人和 NPC 糊在一起像三坨黑土豆。**原因**：对称过度（完美对称像规则几何体缺乏破绽）、负空间不足（胳膊贴身腿并拢成实心块）、道具轮廓融入身体（剑被身体"吞掉"）。**修复**：制造不对称（披风/肩甲/举起的手）；制造负空间（胳膊与身体、腿之间空隙扩大，好的剪影至少有一个"透光的洞"）；道具"跳出来"（武器和身体至少 30° 分叉）。

**错误 2：形状语言混乱——"主角用三角、敌人用圆"。** 玩家战斗中搞不清敌友，或对友好 NPC 本能紧张。**原因**：基本形状和叙事功能不匹配。形状心理学不是玄学 [2]，但靠"剧情说明"覆盖不了玩家潜意识第一反应。**修复**：建"形状分配表"（圆给友方、三角给敌方、方给中立）；角色可混形状但"主导形状"（占轮廓 50% 以上）必须与功能一致；做 L1 练习——朋友不知设定的情况下把反派指认为主角，说明形状语言用反了。

**错误 3：只靠颜色区分——"去色后无法辨认"。** 两个角色用颜色区分（主角红反派蓝），彩色模式没问题，但黑暗场景、色盲模式、远距、夜间模式、黑白画面下全失效。**原因**：颜色是视觉设计中最不可靠的信号之一（练手05 详谈），光照改色、色盲丢色、显示器色域不同——剪影在所有条件下稳定。**修复**：剪影测试加一条"去色测试"（截图转灰度检查关键元素能否区分）；给每角色一个"形状签名"（独特轮廓特征，任何光照下稳定）；不要移除颜色——颜色是增强不是替代。

**错误 4：信息过载——"细节堆满剪影变碎"。** 小尺寸下堆 8 色装饰、每处都加纹理，结果缩小后边缘被吃、不知看哪。**原因**：把"画得丰富"当"画得好"，忽略了可读性是小尺寸第一优先级。**修复**：跑可读性检查四法（100% 实尺 / 模糊 / 他人 / 少即是多）；有疑虑删减而非增加；活下来的是真焦点，被模糊抹掉的是可删装饰。

**自查清单**（每次设计新角色时用）：
- [ ] 填全黑后剪影还有辨识度吗？
- [ ] 缩到游戏内实际像素尺寸后，还能和同场景其他对象区分吗？
- [ ] 剪影中至少一个负空间（透光的洞）？
- [ ] 主导形状（圆/方/三角）和角色叙事功能一致吗？
- [ ] 截图转灰度，关键角色还能互相区分吗？
- [ ] 100% 实尺 + 模糊测试，焦点活下来了吗？

### 2.10 小结

形状是视觉设计中最基础也最被忽视的能力。好消息——它不需要你会画漂亮线条、会配色、懂复杂光影物理。你只需要一个黑色方块和一张白纸，就能训练"形状感知力"。再进一步，五几何体的光影规律 + 五种材质的色板参数，就把"体积"和"质感"两件事的工具箱都给了你。

**记住四件事**：
1. **剪影测试是第一道单元测试**——填黑 + 缩小 + 辨认，通过了再前进。
2. **圆 = 友好，方 = 可靠，三角 = 危险**——不是玄学是进化心理学，选与角色功能匹配的主导形状。
3. **Shape 在先，Form 在后**——先确保轮廓过关，再用五几何体（球/方/柱/锥/环）拆解万物、给它长体积，用材质参数换皮。
4. **可读性是小尺寸第一优先级**——100% 实尺 + 模糊测试，少即是多。

**如果只记住一句话：** 先让剪影过单元测试，再用五几何体给它长体积——细节是最后一步，不是第一步。

**上手行动**：今晚打开练手01 的 `character-v01-outline.aseprite`，花 45 分钟按 L1 给它长一层光影体积。明天醒来缩到 100% 看一眼——它应该从纸片变成一个有厚度的东西。
