# 练手03 空间：留白是画面的肺

### 3.0 这一章解决什么问题

练手02 你给贯穿角色长出了体积——纸片变成了有厚度的东西。但体积只解决"这个东西本身长什么样"，没解决"这个东西周围是什么样"。你把角色往场景里一放，画面还是"不对劲"——不是角色画错了，是**角色周围的空间你没安排**。这一章解决：**怎么安排画面里的"空"——留白、间距、纵深——让它从"一堆东西摆在那"变成"一个让人想多看两眼的场景"。**

观察02 把"空间"列为八种数据类型之一，给了你一句话定义和两个极端：《Loop Hero》一条环路被大片空地包围，负空间营造冥想感；《Undertale》弹幕框塞满弹幕 = 压迫。这一章把它彻底拆开三件事：空间到底是什么（CSS 盒模型类比会让你瞬间开窍）、留白的几种"配比"各传达什么情绪、怎么在 2D 像素画面里用五层纵深把平面变立体。

这一章是 A 原声——书 A 的"空间"章几乎原样搬来，只把非像素的例子换成像素游戏。它和练手02 互为表里：练手02 讲"实"的形状体积，练手03 讲"虚"的留白纵深，两章合起来你才有完整画面骨架——有体量的人物站在有深度的场景里。**贯穿式角色项目暂停一程**：这章练场景，角色在**练手04（明度）**深化明暗、**练手05（色彩）**上色、**制作02（像素角色工作流）**收口；本章的视差分层和呼吸点在**制作03（环境与 Tile）**落地成 tile 场景，负空间与视觉权重在**练手07（构图）**正式收进构图法则。

你只需要 Aseprite，加一支笔一张纸。

### 3.1 程序员视角：空间到底是什么

#### 3.1.1 CSS 盒模型——你已经在用了

如果本书只能有一个程序员类比，这一节的类比就是它。

打开浏览器的开发者工具，选中页面上的任何一个 `<div>`，都会看到右下角出现一个叠了四层的彩色方块。你看到的这四层，就是空间的全部秘密：

```
┌─────────────────────────────┐
│         margin              │  ← 元素之间的空隙（外部间距）
│   ┌─────────────────────┐   │
│   │      border         │   │  ← 可见边界
│   │   ┌─────────────┐   │   │
│   │   │  padding    │   │   │  ← 内容和边界的空隙（内部间距）
│   │   │ ┌───────┐   │   │   │
│   │   │ │content │   │   │   │  ← 文字、图片、按钮
│   │   │ └───────┘   │   │   │
│   │   └─────────────┘   │   │
│   └─────────────────────┘   │
└─────────────────────────────┘
```

**content = 正形（Positive Space）**——画面中"有东西"的部分：角色、建筑、UI 按钮、一棵树。

**margin + padding = 负形（Negative Space）**——画面中"什么都没有"或"东西很少"的部分：天空、地面空白、角色之间的空隙、UI 面板和屏幕边缘的距离。

现在回忆你写前端时那些"说不清哪里不对"的页面。十有八九是盒模型出了问题：`margin: 0` 的卡片列表挤成一团，`padding: 0` 的按钮文字贴着边框像要爆炸，`line-height: 1` 的段落行贴行让人透不过气。**你把 CSS 规则替换成视觉规则，问题一模一样：**

| CSS 症状 | 视觉症状 | 结果 |
|---------|---------|------|
| `margin: 0` everywhere | 画面元素全挤在一起 | 玩家"憋得慌" |
| `padding: 0` | UI 文字贴着面板边缘 | 廉价感、不专业 |
| `margin: 100px` everywhere | 画面空旷到散架 | 没有视觉中心 |
| `margin: 0 auto` 给了一个合适的值 | 关键元素居中，四周有呼吸感 | 经典"看起来舒服" |

**你不需要学"绘画的空间感"。你需要做的就是在画布上给每个元素写 `margin` 和 `padding`。只不过这行 CSS 不在代码里——它在你的构图决策里。**

#### 3.1.2 空间在 2D 游戏中的双重身份

在平面设计（UI、书籍排版）里，空间只做一件事：界定正形和负形的关系。但在 2D 游戏中，空间有两个完全不同的身份：

**身份一：平面留白（Flat Space）**。你的战斗 UI 里技能图标之间的间距、对话框和屏幕边缘的距离、角色和地面之间的空隙。这些是你作为玩家"不会看但会感觉"的东西——就像你浏览网页时不会注意 `margin` 的数值，但如果它消失了，你第一时间就会发现。

**身份二：纵深空间（Depth Space）**。在 2D 游戏中，纵深不是真实存在的——你的屏幕是平的。但你仍然需要制造"前后"的错觉。这层错觉叫 **Z-depth**，而实现它的手段——视差滚动、大气透视、色彩后退、细节衰减、叠压——本质上都在做一件事：**给每一层元素分配不同的空间值。**

把它翻译成你熟悉的语言：**纵深 = 不同 `z-index` 的 `<div>` 以不同速度滚动。** 远山的 `z-index: 1` 滚动速度 = `transform: translateX(-1px)`，前景树的 `z-index: 10` 滚动速度 = `transform: translateX(-5px)`。两者一叠加，人脑自动解读为"远山在后，树在前"。这就是视差滚动的全部秘密——**你不是骗了眼睛，你是在喂给大脑它本来就认识的空间线索。**

> **程序员类比关键总结：**
> - **平面留白 = margin + padding**：控制元素间距和边界呼吸，决定画面"松紧度"。
> - **纵深空间 = z-index 分层 + 不同 transform 速度**：控制深度层次，决定画面"远近感"。
> - **正形 = 有 content 的元素（有填充的 div）**。
> - **负形 = margin/padding 区域（无内容的空间）**。

这两种身份在你的游戏里是同时存在的。一个 2D 场景同时需要"平面上的留白舒服"和"纵深上的层次清楚"。任何一个出了问题，画面就"不对劲"。

### 3.2 理论最小集

#### 3.2.1 正形与负形——画画先空后满

十九世纪末，美国艺术教育家 Arthur Wesley Dow 说过一句话，被后世无数设计师引用："空白是绘画的肺"（Space is the lungs of a picture）[2]。你用肺吸气进来——这是往画面上加元素；你用肺呼气出去——这是在画面上留出空白。只吸不呼，人会窒息。只往画面上加东西不加空白，画面会"憋死"玩家。

Dow 的理论核心极其简单：**一幅画不是"填满一块布"，而是"在一块空间里安排形状"。** 新手的第一反应永远是"空着的地方怎么办？加点什么！"——和程序员的第一反应"空白页面？写点代码！"一样。但 Dow 会说：空着的地方不是你失败的地方，空着的地方是你主动留出来的。

![正形与负形——画面的"内容"和"肺"](images/练手/练手03-fig01-正负形.svg)

*图 练手03.1：左侧为原始场景，右侧用红色标注正形（有内容的元素）、蓝色标注负形（留白/空间）。同一个场景，标注后你立刻能看到空间是如何"撑起"画面的。*

在游戏美术里，正形和负形承担不同的功能：
- **正形**：承载信息——角色是谁、Boss 在哪、门往哪开。
- **负形**：决定信息的读取体验——场景是紧张还是放松、UI 是专业还是廉价、玩家是先看主角还是先看背景。

#### 3.2.2 正负形五比例实验——同样的元素，不同的情绪

这可能是全书性价比最高的一个实验：同样的白色方块，放在不同的黑色画框里，**只因正负形比例不同，传达的情绪完全不同。** 这不是艺术家的主观感受——它源自阿恩海姆（Rudolf Arnheim）在《艺术与视知觉》[3] 中对视觉平衡的系统研究：人眼天生会把画面中的元素和空白视为一个力学系统，不平衡的比例会让大脑产生"紧张感"或"压迫感"。

![正负形五比例实验——同一元素，不同空间比例，不同情绪](images/练手/练手03-fig02-五比例实验.svg)

*图 练手03.2：五组正负形比例实验。同一元素、不同空间安排，传达完全不同的情绪。*

| 正形:负形 | 情绪 | 游戏场景适配 | 你的感觉 |
|----------|------|-------------|---------|
| **50:50** | 稳定、均衡、但平庸 | 一般对话场景、商店界面 | "还行，没什么特别" |
| **30:70** | 紧张、对比强、视觉重量感 | Boss 出场、关键剧情 | "被盯着看" |
| **70:30** | 压迫、拥挤、窒息 | 狭窄的地牢、末日场景 | "想逃离这里" |
| **10:90** | 空旷、孤独、渺小 | 沙漠独行、宇宙漂流 | "我很小，世界很大" |
| **90:10** | 膨胀、溢出、恐怖 | 特写面部、关键物品 | "它太大了，要吞了我" |

《Hyper Light Drifter》里最经典的反差：漂泊者独自走在广袤废土上的片段，正负形比例约 10:90——你作为玩家清晰地感受到自己的渺小。几秒钟后，一座 Titan 巨像在画面中浮现，比例骤变成 30:70——比例变化本身比任何 HP 数字都更能告诉你"你打不过它"[4]。同样是像素画面，没有一句台词，空间比例替你完成了叙事。

具体的游戏场景适配：

| 场景类型 | 推荐正负形比 | 同理类比 |
|---------|-----------|---------|
| Boss 战画面 | 30:70（Boss 占 30%） | 给 Boss 留出 70% 的"威胁空间" |
| 开放世界探索 | 15:85（角色占 15%） | 世界必须"大"——大意味着你的留白大 |
| 地牢/室内 | 60:40（墙壁占 60%） | 狭窄感来自四壁压过来——正形主导 |
| UI 对话面板 | 40:60（面板占 40%） | 文字周围的 padding 至少 30px |
| 标题画面 | 20:80（标题占 20%） | 一个单词 + 大片天空 = AAA 感 |

#### 3.2.3 2D 游戏纵深五层技术全景

平面留白解决了"松紧"，纵深层次解决了"前后"。在 2D 像素游戏里制造纵深错觉，不需要 3D 建模，只需要掌握这五层技术——它们按"底层实现→高层感觉"排列，越往下越基础，越往上越细腻：

![2D游戏纵深五层技术全景](images/练手/练手03-fig03-纵深五层.svg)

*图 练手03.3：2D游戏制造纵深错觉的五层技术。它们是叠加使用的——只用叠压关系效果弱，五层全上效果强。*

**第一层：叠压关系（Occlusion / Overlap）——最基础、最不需要学习**

这是你三岁时就掌握的深度线索：一个东西挡住另一个东西，被挡住的那个更远。在 2D 游戏里，前景物体必须挡住中景物体，中景必须挡住远景物体。这规则简单到你可能觉得不用提——但程序员做 UI 经常违反叠压：让一个对话框"在"角色前面，对话框后面却露出角色一角，那一角就在告诉大脑"对话框在角色后面"，你肉眼不会注意但大脑会默默把场景标记为"奇怪"。操作规则：渲染顺序 = 远 → 中 → 近，远景先画、中景覆盖远景、前景覆盖中景——一帧里叠压出错（比如远景树冠"跨过"中景建筑），大脑就收到互相矛盾的深度信号，产生眩晕感。

**第二层：视差滚动（Parallax Scrolling）——你会写代码就能做**

视差滚动的本质你已经知道了：远层比近层移动得慢。数学公式极其简单：

```
// 每一帧的 x 偏移量
layer[N].offsetX = camera.offsetX * layer[N].parallaxFactor;

// 各层的 parallaxFactor 典型值：
//   远景天空:  0.0  （完全不动，如月亮）
//   远景山:    0.2  （极慢）
//   中景树:    0.5  （中等速度）
//   近景路面:  1.0  （跟随相机）
//   前景草:    1.2  （比相机还快，制造"很近"的错觉）
```

《Celeste》的场景使用了 4-6 层 parallax [5]。这不是炫技——你站在游戏中往前跑时，这些层以不同速率后退，你的大脑才"相信"自己在一个有深度的世界里。如果你只有一层背景图（parallaxFactor = 0.0），世界看起来就像是一个在跑步机上滚动的壁纸。

**第三层：大气透视（Atmospheric Perspective）——远景加蓝加灰**

真实世界中，你和远处山之间隔了数公里的空气，空气中悬浮的水汽和微粒会散射蓝光——所以远山看起来偏蓝、偏灰、对比度很低。你不需要精确的 RGB 数值，只需记住"每往远一层，加一层蓝色半透明蒙版"——在 Aseprite 里给远景层新建一个 10%-20% 不透明度的蓝色填充层覆盖上去即可。2D 像素画里的色阶参考：

| 距离 | RGB 调整 | 对比度 | 饱和度 |
|-----|---------|-------|-------|
| 近景 | 原色 | 100% | 100% |
| 中景 | +5% 蓝通道 | 80% | 80% |
| 远景 | +10% 蓝通道 + 整体加灰 | 50% | 50% |
| 最远景 | 几乎纯蓝灰色 | 20% | 20% |

**第四层：色彩后退（Color Recession）——暖进冷退**

暖色（红、橙、黄）在视觉上有"前倾感"，冷色（蓝、青、紫）有"后退感"。这不是心理学的推论，而是光学物理的副产品——暖色光（长波）在眼睛里的成像面略靠前，冷色光（短波）成像面略靠后 [6]。

游戏里怎么用：前景主角/敌人的核心区域暖色调（即使冷色调游戏也相对更暖）、中景中性、远景冷。把色彩后退和大气透视叠加——远景同时变冷变灰——纵深效果指数级放大；反过来远景比近景还鲜艳（常见初学者错误），纵深当场就崩了。

**第五层：细节衰减（Detail Decay）——远近有别**

远处的树不需要画每一片叶子，远处的角色不需要画每一根头发——真实世界距离越远细节越少。像素画里这对应一个简单规则：近景完整纹理/多色渐变/精细轮廓，中景简化纹理/少色块面/清晰轮廓，远景纯色块面/只有轮廓/无纹理。细节衰减和你做 LOD（Level of Detail）3D 渲染是同一逻辑——只不过在 2D 里手动实现：远处用更少颜色、更大色块、更低对比度。这也是像素画天然擅长的：低分辨率远景本就得用大色块，把"被迫的简化"变成"主动的纵深线索"。

#### 3.2.4 呼吸点——每个场景必须有一个"什么都没有"的区域

这是本章最重要的一个操作性原则。在 3.0 里我们提到《Loop Hero》60%-70% 的画面是空地。这不是偶然——这是刻意为之。

![呼吸点——让眼睛有地方休息](images/练手/练手03-fig04-呼吸点.svg)

*图 练手03.4：一个场景中的呼吸点（绿色虚线框）。这个区域元素最少、对比度最低——不是"没画完"，是"故意空着"。*

**呼吸点（Breathing Point）**的定义：画面中至少一个"元素最少、对比度最低、色彩最接近背景"的区域。它不是画面的"主题"——主题在那 80% 的热闹区域里。它是画面的"肺部"——在那 20% 的安静区域里。

没有呼吸点的画面 = 拿着喇叭在你耳边连续喊三分钟。观众的瞳孔会持续处于高负荷状态，而他不知道自己为什么不想看了——他只是直觉上"想换个界面"。反过来，《Downwell》把整个竖井下落画面压成"中央角色 + 大片干净背景"，几乎每一帧都给了眼球一个明确的休息区——这是它能在高速动作下还保持"清晰不乱"的关键。

在你的游戏里找呼吸点：UI 界面面板和屏幕边缘之间至少留 20px 净空；战斗场景画面右下角或左上角至少有一个纯色/渐变区域不被任何 UI 或角色占用；关卡场景至少留一块完整天空——山脉和建筑不把它完全切断。

> **程序员类比：呼吸点就像代码里的空行和缩进。** 你在写一个复杂函数时，如果 200 行代码没有任何空行、没有任何缩进变化——你能读懂吗？能，但你需要像侦探一样一个一个 token 扫描。如果你在逻辑段落之间加一个空行，你的大脑就能在 0.1 秒内分辨"这块是一段逻辑"。视觉呼吸点做的正是这件事——**它给眼球提供段落分界符。**

### 3.3 练习

本章三个练习按"识别→体验→创作"递进。L1 只动眼，L2 动手但不需要工具，L3 动手且产出小作品。每个练习都有清晰的合格标准——做到了才算通过。三个练习建议分三天。

#### L1 · 正负形标注练习（约 15 分钟）

**目标**：建立"一眼看空间比例"的能力。你不需要任何绘画工具——只需要你的眼睛和截图。

**步骤**：

1. **找 5 张截图**。打开 Steam/App Store/Google Images/Bing Images（优先使用高清、无水印截图来源），找 5 张你感兴趣的游戏截图。按以下类别各找一张：
   - 一张"看起来舒服"的像素游戏截图（如《Loop Hero》《星露谷物语》的风景帧）
   - 一张"看起来很挤"的游戏截图（如复杂 UI 的弹幕射击或策略游戏）
   - 一张"看起来很空"的截图（如极简 1-bit 独立游戏《Minit》或开放世界风景）
   - 一张高制作水准大作截图（作对比基线——《空洞骑士》《茶杯头》级别）
   - 一张独立像素游戏战斗/动作截图（《Celeste》《Dead Cells》《Downwell》等）

2. **标注正形和负形**。对于每一张截图，你可以直接在手机上用手指在截图上画圈，也可以用任何图片查看器的标记功能，或者干脆打印出来用彩笔——重点不是工具，是**真的做标注，不是"我看一眼就行了"**。用红圈标出画面中正形最集中的区域（元素密集区），用蓝圈标出负形（留白最多的区域）。

3. **目测比例并记录**。给每张图目测正负形比例，填一张小表：

| 截图 | 来源游戏 | 你目测的正负形比 | 传什么情绪（紧张/放松/压迫/空旷） | 有呼吸点吗？在哪？ |

**L1 合格标准**：
- [ ] 5 张截图全部完成标注（红圈正形、蓝圈负形）
- [ ] 表格全部填完且比例目测偏差不超过 ±15%（"舒服"的截图不会是 50:50，开放/探索类截图常见 15:85~30:70）
- [ ] 至少一张截图中找到了呼吸点
- [ ] 你能说出"拥挤"和"舒适"在空间比例上的差异（不会低于 2 句话）

#### L2 · 正负形五比例实验（30-60 分钟）

**目标**：亲自体验 3.2.2 中的五组正负形比例对情绪的影响。这个练习不需要任何画图技巧——你只用一个方块和一个画框。

**步骤**：

1. **准备画布**。找一张白纸（或打开任何画图软件的新画布），画五个等大的正方形框，每个边长约 10cm（或 400px）。这五个框就是你的五块"画面"。

2. **填黑色**。把五个方框内部全部填黑（或用黑色笔涂黑，或用油漆桶工具填黑）。

3. **放白色方块**。在五个黑色方框里各放一个白色方块，但大小不同：
   - 框 1：方块占框面积的约 50%
   - 框 2：方块占框面积的约 30%
   - 框 3：方块占框面积的约 70%
   - 框 4：方块占框面积的约 10%
   - 框 5：方块占框面积的约 90%

   注意：每个白色方块放在框的**不同位置**——不要全居中。比如框 2 的方块放右上角，框 3 的放左下角——位置也会影响情绪，你要感受这个影响。

4. **记录感受**。盯着每一个方框看 10 秒，写下第一感受——不是"好看"或"不好看"，是**身体感受到的情绪**："觉得透不过气""感觉它在盯着我""很安静""想往右上看"等。

5. **复现游戏场景**。选一个你最喜欢的像素游戏，用一个简单的几何体来代表它一个典型场景的正负形比例。比如《Loop Hero》是一条小环路占画面约 15%，其余是空地——你只需要画一个黑框 + 一个小白圆环。然后问自己：这个游戏为什么选择这个比例？

| 你的第几个框 | 正负形比例（目测） | 你的第一直觉感受 | 让你联想到哪个游戏/场景？ |
|------------|----------------|---------------|---------------------|
| 框1 | 50:50 | | |
| 框2 | 30:70 | | |
| 框3 | 70:30 | | |
| 框4 | 10:90 | | |
| 框5 | 90:10 | | |

**L2 合格标准**：
- [ ] 五个框全部完成且方块大小基本符合目标比例（目测偏差不超过 15%）
- [ ] 表格全部填写完毕，每个格至少有 5 个字
- [ ] 10:90 和 90:10 两组的感受差异明显且不同——如果你觉得"都差不多"，回头重新看一次
- [ ] 你能回答这个关键问题："我做的游戏中，哪个场景用哪种正负形比例最合适？"（至少写出一句话答案）

#### L3 · 三层色块风景——只靠空间表达深度（1-3 小时）

**目标**：只用纯色色块（不用任何线条！）构建一个有纵深感的简单像素风景，证明——你只要控制空间，不需要会画任何东西。**你需要**：Aseprite（或任何能做图层 + 填色的工具：Figma、Photoshop、Krita、Procreate 甚至 Google Slides/PowerPoint），只要支持"创建图层 + 画矩形/椭圆 + 填色"就行。

**步骤**：

1. **准备画布**。新建画布，16:9 横版（推荐 160×90 或 320×180 像素——刻意低分辨率，逼你用大色块）。

2. **建立三层图层**。从下到上建三个图层：
   - 图层 1（最底）· 远景：天空和远山
   - 图层 2（中间）· 中景：中距离的山或建筑群
   - 图层 3（最顶）· 近景：前景地面、一棵树或一块石头

3. **远景层（只用纯色！）**——天空用淡蓝渐变或纯色填充（#C8E0F0 或类似），远山用比天空深的蓝色（#A0B8D0），轮廓简化——远山的"山形"只需要 3-5 个点的简单多边形。对比度保持最低：远山和天空的亮度差不要太大。

4. **中景层**——中景山用比远山更深的蓝色或绿色（#7090A0 或 #5A7A40），轮廓可以比远山稍微复杂一点（6-8 个点的多边形），但不要画任何树木细节——纯色块只靠形状。

5. **近景层**——近景用最饱和、最暖的颜色（暗绿 #3A5A20 或暖棕 #6B4A30）。前景元素可以更大，但**不要超过画面的 30%**——给远景留出至少 70% 的空间。如果你需要放一棵树：一个棕色矩形当树干 + 一个深绿色大椭圆当树冠——两刀到位，不要画树叶。

6. **加一个呼吸点**——在画面右下角或左上角故意"不画任何东西"。这个区域只有天空色。它的大小应占画面至少 15%。

7. **检查五层纵深技术**。逐一对照 3.2.3 的五层技术：
   - [ ] 近景是否挡住中景？（叠压关系 ✓）
   - [ ] 远景颜色是否比近景偏蓝/偏灰？（大气透视 ✓）
   - [ ] 近景颜色是否比远景更暖？（色彩后退 ✓）
   - [ ] 远景形状是否比近景简化？（细节衰减 ✓）
   - [ ] 是否有至少一个呼吸点？（呼吸点 ✓）

**L3 合格标准**：
- [ ] 至少三层可辨识的纵深（远景/中景/近景），不使用任何线条
- [ ] 五层纵深检查全部打勾（✓）
- [ ] 呼吸点面积 ≥ 画面的 15%
- [ ] 正负形比例在 15:85 ~ 30:70 之间（"舒适的开放空间"，此为推荐而非硬性标准）
- [ ] 你能不借助任何参考，用文字描述你的这幅场景中"每一层的 RGB 色相是如何支持纵深感的"（比如："近景绿色中微加黄（暖），远景蓝色中加了白（冷+灰）"）

**如果 L3 太难，降级版**：只用两层——一层天空（远景）+ 一层地面 + 一个方块（近景）。只要能靠纯色块表达"那个方块离我近，天空离我远"，就算通过。

### 3.4 常见错误与诊断

**错误 1：正负形比例单调（总是 50:50）——"不松不紧但平庸"。** 每个场景、每个 UI 面板、每个关卡正负形都对半开，玩家反馈不是"难看"是"没感觉"。**原因**：你没把空间比例当"情绪参数"而是当"默认设置"——50:50 是所有视觉工具的出厂值，不主动调它自然就是这个值。**修复**：打开任意截图用 L1 方法目测比例；按叙事目标调——"放松探索"→15:85~25:75、"紧张战斗"→30:70（Boss 占 30%）、"压迫窒息"→60:40；改了再往回退一点，跟调 API 超时一个逻辑，先推到极限再微调。

**错误 2：没有呼吸点（画面每个角落都塞满）——"热闹但眼睛累"。** 玩家说"看了几分钟就想休息"，可每个元素单独看都没问题。**原因**：背景装饰、粒子、UI 图标、状态条、角落花纹组合起来就是"所有像素都在喊"，没有安静区让眼球休息——就像一段代码没空行没缩进，编译器能跑人不能读。**修复**：打开截图找一个"纯色"区域，找不到就确诊了；强制删掉一个角落的所有元素给出完整空区；呼吸点至少占画面 10%（《Loop Hero》用 60%-70%，你不必那么激进，10% 是底线）。

**错误 3：纵深关系混乱（远景比近景还鲜艳/饱和）——"有三层但看着晕"。** 用 L3 五项检查发现远景山的蓝比近景树的绿还饱和。**原因**：违反色彩后退和大气透视——你把各层颜色当"独立配色"各选好看的，组合起来互相打架，大脑收到"这不应该发生"的信号标为"奇怪"。**修复**：远景饱和度必须 ≤ 近景的 50%（复制近景色板加 10%-20% 白+蓝覆盖即远景色）；不确定就把远景转灰度，明度必须 远景 ≥ 中景 ≥ 近景，越远越亮/灰——拿手机拍窗外远山验证，远山一定更灰更蓝。

**错误 4：叠压关系前后颠倒——"中景树枝在角色前，角色脚又在树枝上"。** 这种前后矛盾让大脑产生微妙失控感。**原因**：渲染顺序不对或图层管理混乱——先画角色后随手加树枝，树枝图层在角色后却在画面跨过角色一条腿，相当于给两个元素设了互相矛盾的 z-index。**修复**：严守渲染顺序 远景→中景→角色→前景→UI，后面的层永远不覆盖前面的层；要"树枝在角色前"就把树枝切到角色层之上（Aseprite 里改图层）；用 L3 叠压检查清单，近景树冠有任何像素"跨过"中景山就修掉。

### 3.5 小结

空间不是画完之后"剩下的部分"——空间是你画画时第一个要做的决策。一个画面的呼吸感、紧张感、压迫感、空旷感，都是正负形比例的外在表现，而不是"颜色调得好"或"线条画得好"的结果。

**关键三件事**：
1. **正形 = content，负形 = margin+padding**。盒模型是你在任何二维设计中控制空间的母语。
2. **正负形比是情绪参数**——10:90（孤独）→ 30:70（紧张）→ 50:50（稳定）→ 70:30（压迫）→ 90:10（窒息）。选择比例就是选择玩家看画面的第一反应。
3. **2D 游戏纵深 = 五层技术叠加**——叠压 + 视差 + 大气透视 + 色彩后退 + 细节衰减。单用哪一层效果都有限，全用才"真实"。

**如果只记住一句话：** 先在画布上给每个元素写 margin 和 padding，再动笔——留白是画面的肺，不是画完之后剩下的边角料。

**上手行动**：今晚随便挑你游戏里一个你"觉得不对"的场景截图，用 L1 的方法标一遍正负形、目测比例、找呼吸点。十有八九你会发现：不是画工问题，是 `margin` 没写够。

### 3.6 扩展阅读

1. **《Celeste》开发者笔记与 GDC 演讲**（Maddy Thorson / Noel Berry）——Celeste 的视差分层、场景节奏、像素美术迭代过程在 Thorson 的开发博客和 GDC 演讲中有大量讨论，是"像素游戏里怎么做纵深"的一手资料。推荐指数：★★★★★
2. **《Art and Visual Perception》**（Rudolf Arnheim, 1974, University of California Press）——第 1 章"平衡"和第 9 章"空间"对正负形的讨论是本章的理论根基。如果你只读一本设计理论书，这本。推荐指数：★★★★★
3. **《CSS: The Definitive Guide》盒模型章节**（Eric Meyer & Estelle Weyl, O'Reilly）——不，这真不是玩笑。CSS 文档是你理解平面空间的最实用入口。margin/padding/border/content 这四个概念一旦在你脑子里固化了，你看任何游戏画面都会自动做空间分析。推荐指数：★★★★☆
4. **《The Art of Game Design: A Book of Lenses》透镜 #14"空间透镜"**（Jesse Schell, 3rd ed., 2019, CRC Press）——Schell 用一页纸讲清楚了你为什么需要在关卡设计中"主动"控制空间密度。和他所有的透镜一样，这篇极短但极有力。推荐指数：★★★★☆
5. **Drawabox 第 1 课——关于"空间中的形状"的练习**（Irshad Karim, drawabox.com）——这里的"空间"是 3D 中的物体位置关系而非平面留白，但它对正形/负形感知的训练极有效。完成第 1 课的前三组练习后，你的眼睛会开始不自觉地做正负形分析——而这正是 L1 练习要的效果。推荐指数：★★★★★
