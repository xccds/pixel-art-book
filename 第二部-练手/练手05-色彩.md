# 练手05 色彩：少即是多

### 5.0 这一章解决什么问题

练手04 你给贯穿角色建好了明度骨架——`character-v03-value.aseprite`，去色后焦点还在、层次还清。这一章解决**从明度骨架到彩色皮肤的过渡**：调色板怎么选？阴影是直接加黑还是偏个色？为什么精心挑的"好看的蓝紫色"涂上去画面就变脏？

观察02 给了你一条黄金规则——**色彩在游戏中的第一作用是信号，第二作用才是装饰。**《蔚蓝》冷紫标危险、暖橙标安全；《星露谷》靠黄昏暖色建立怀旧情绪。这一章把这条规则彻底拆开：色彩为什么是信号、为什么少即是多、着色选西式还是东式、色域怎么限定、调色板怎么拿来即用。

> **贯穿式角色项目（续）：** 练手01 你画了 `character-v01-outline.aseprite` 的剪影/线稿，练手04 L1 用明度思维打光存为 `character-v03-value.aseprite`。**本章 L1 给这个角色正式上色**——套一套有限调色板 + 色相偏移着色，存成 `character-v04-color.aseprite`。它将在**制作02（像素角色工作流）**收口成可导入引擎的完整角色。

### 5.1 色彩功能分层 + 少色原则

色彩在游戏中的第一身份是**功能载体**，第二才是审美。把每个颜色按功能归类：

- **交互层——最高优先级**：危险提示（红闪）、选中状态（黄框）、可交互高亮（白晕）。《蔚蓝》冷紫标危险区、暖橙标安全平台，玩家不读字就懂。
- **信息层**：阵营识别（红/蓝）、稀有度（白/绿/蓝/紫/橙）、属性类型（火=红、冰=蓝）、状态标识（中毒=绿框）。
- **情绪层**：暖色调→安全/怀旧，冷色调→孤独/危险，低饱和→沉重。Gris 每章一个主色定情绪。
- **装饰层——最低、最可被覆盖**：天空色、草木色、环境氛围色。

![色彩功能分层金字塔](../images/练手/练手05-fig01-色彩功能金字塔.svg)

*图 练手05.1：色彩功能分层（交互 > 信息 > 情绪 > 装饰）。*

> **程序员类比 #1：** 色彩功能分层 = `z-index` 系统。交互层是 `z-index: 9999`，信息层是 `struct`/`enum`，情绪层是 `theme`，装饰层是 `style`。让 CSS 覆盖核心逻辑的输出——画面好看，但玩家不知道怎么玩。

**少色原则。** 如果 4 色能表达清楚一个场景，用 16 色就是过度设计。Game Boy 4 级灰度出了《宝可梦》，8-bit 有限色板出了《超级马力欧》（蓝的天、绿的管、红的马力欧，三主色全球通识）。现代独立像素通常 8–16 色一个场景。每加一个色你要付三个成本：**注意力成本**（新色发声，主角信号被稀释）、**和谐成本**（色越多越难协调）、**维护成本**（多场景多光照下难保一致）。

**有限调色板系统——不同色数能做什么：**

| 色数 | 分工 | 范例 | 适用 |
|------|------|------|------|
| 3 色 | 1背景+1中间+1高亮 | Limbo 黑白灰 | 极简、原型、Game Jam |
| 4 色 | 1背景+2角色+1高亮 | Game Boy 时代 | 最小原型、像素入门 |
| 8 色 | 天空+地面+角色(2)+植被+建筑+UI+暗部 | 《蔚蓝》单区域 | ⭐2D独立游戏甜点 |
| 16 色 | 每主色 2-3 明度变体+2 中性灰 | 《空洞骑士》每区 | 像素场景上限，超易脏 |

> **程序员类比 #2：** 少色原则 = **最小权限原则**。一个函数只应拥有完成任务所需的最小权限，一个画面只应拥有表达信息所需的最少颜色。色数 = `enum` 的大小——3/4/8/16 按项目规模选。

### 5.2 HSV 三轴——程序员的色彩语言

屏幕颜色由 R/G/B 三通道混合，hex 是它的内存地址——机器友好，人不友好。**HSV 才是你选色的原生语言**：

- **Hue 色相**（0°–360°）：什么颜色。红=0°、黄=60°、绿=120°、蓝=240°、紫=270°。
- **Saturation 饱和度**（0%–100%）：有多纯。100%=纯色，0%=灰度。降低饱和度是让颜色变灰的唯一方式——不是加白（那是提明度），不是加黑（那是降明度）。
- **Value 明度**（0%–100%）：有多亮。练手04 的全部内容。

![HSV色彩模型简图](../images/练手/练手05-fig02-HSV模型.svg)

*图 练手05.2：HSV 三轴。三个参数可独立操控。*

三个参数的独立操控是你的超能力：**换颜色保持结构**——只改 H，不动 S/V；**调氛围**——不改 H，整体降 S + 调 V；**做阴影**——V − 20%，S + 10%，H 向冷色偏移 5–15°（5.3 展开）。

> **程序员类比 #3：** RGB/hex = 颜色的内存地址（指针），HSV = 颜色的语义化字段（`struct {hue, sat, val}`）。选色用 HSV 三轴思考"更冷/更灰/更亮"，比手调三个 0–255 通道直观一个量级。

### 5.3 着色法：东西方流派 + 色彩斜坡 + 色偏阴影

同一个物体，从亮面到暗面怎么过渡颜色？两种哲学：

**西式——逐级明度阶梯。** 保持色相不变，只通过加减明度调节深浅。一个橙色球：高光 `#FFD9A0` → 亮面 `#FFB870` → 中间调 `#E09040` → 暗面 `#B56820` → 阴影 `#7A4010`——**全是同一橙色相，只 V 在变**。简单直接，适合入门。

**东式——色相偏移（Hue Shifting）。** 调明度的同时**改变色相**：高光往暖色（黄方向）偏，阴影往冷色（蓝紫方向）偏。同一个球：高光 `#FFE890`（暖黄）→ 亮面 `#FFC060`（暖橙）→ 中间调 `#E09040`（橙）→ 暗面 `#9A5868`（紫红）→ 阴影 `#5A3866`（冷紫）。色相沿色轮从黄滑向紫。许多专业像素艺术家偏好东式——它产生更丰富、更有生命力的色彩。

![东西方着色对比](../images/练手/练手05-fig08-东西方着色对比.svg)

*图 练手05.8：左·西式（色相不变，只调 V）；右·东式（暖高光→冷阴影，H 沿色轮滑动）。*

**色彩斜坡（Color Ramp）。** 一组按明度排列的颜色，从亮面到暗面的过渡，通常 4–6 个梯级。上面两个球的 5 个色就是一个 ramp。创建步骤：选基色 → 复制到相邻色板 → 调明度和色相 → 在两端之间生成过渡。这是你给物体上色的标准数据结构。

> **程序员类比 #4：** 色相偏移 = **给 ramp 加 HSL 空间的曲线插值，而不是线性灰阶插值**。西式 ramp 是 `lerp(light, dark, t)`——同色相的灰阶；东式 ramp 是 `lerp_hsl(warmHighlight, coolShadow, t)`——V 降的同时 H 向冷色偏，像给数值加了一个相位偏移。暗部往冷偏、亮部往暖偏，模拟的是真实光的环境散射（蓝天反射填冷光进阴影），所以有空气感而不像在煤灰里滚过。

**色偏阴影（Color-Biased Shadows）——别再加黑了。** 初学者画阴影：选物体色 → 加黑色。这是错的。自然界阴影被环境光填充，而环境光通常带冷色调（蓝天反射的蓝光）。阴影不是"变黑"，是"变冷 + 略暗"——这就是东式的色相偏移。Gurney 的原则：**暖光（阳光/灯光）→ 冷影（加蓝/紫偏移）；冷光（阴天）→ 暖影（加橙/棕偏移）。**

**实操公式（背这一个就够了）：** `阴影色 = 基础色 + V 降 15-25% + S 略升 5-10% + H 向冷色（蓝紫方向）偏 5-15°`。高光反向：H 向暖色（黄方向）偏 5-10°。阴影立刻有空气感——这不是魔法，是大气散射的物理。

### 5.4 选色范围：色域遮罩 + 六和谐方案

怎么把不同颜色组合在一起还好看？两套工具，同一枚硬币的两面：

**色域遮罩（Gamut Mask）——管"禁止什么"。** James Gurney 的方法：在色相环上画一个三角形作为"可用色域"。三角形内的高饱和色自由用，三角形外的色要么不用、要么以极低饱和（灰化）出现。画秋天：遮罩覆盖红-橙-黄，画面所有高饱和色都来自这三个色相；你仍可用蓝——但只能是灰蓝做天空淡化。结果：整张画由暖色"家族"主导，自动获得和谐。

常见遮罩类型：**类比色**（覆盖相邻 2–3 色相，最和谐，适合宁静场景）、**互补色**（跨 180°，高张力，适合角色弹出背景）、**三角形**（三色均布，活泼平衡）。

![色域遮罩](../images/练手/练手05-fig04-色域遮罩.svg)

*图 练手05.4：色域遮罩。在色相环上画三角形作为可用色域——域内高饱和自由用，域外只能灰化。*

> **程序员类比 #5：** gamut mask = **类型系统的约束**。把可用色限定在色相环的子集内，就像把变量限定在一个 `enum` 内。约束反而让画面更安全、更可预测、更一致——静态类型检查让代码更稳定，色域遮罩让画面更和谐。

**六和谐方案——管"取什么"。** 色轮上不同取色几何产生不同效果：

| 方案 | 色相数 | 对比度 | 和谐度 | 推荐用途 |
|------|--------|--------|--------|---------|
| 单色 Monochromatic | 1 | 极低 | 极高 | 入门、氛围统一 |
| 类比 Analogous | 2–3 | 低 | 高 | 环境/背景/场景 |
| 互补 Complementary | 2 | 极高 | 中 | 角色突出、UI 强调 |
| 分裂互补 Split-Comp | 3 | 中高 | 高 | 精致插画、游戏场景 |
| 三角 Triadic | 3 | 中 | 中 | 多彩角色、幻想 |
| 四角 Tetradic | 4 | 极高 | 低 | 复杂场景（高级） |

![六和谐方案速查](../images/练手/练手05-fig09-六和谐方案速查.svg)

*图 练手05.9：六种和谐方案——不同取色几何。*

要点：**互补色永远不要五五开**——70/30 或 90/10，主色占大头、补色只做点缀。**60-30-10 法则**：主导色 60-70%（角色/背景主调）+ 辅色 20-30%（次要元素）+ 点缀色 5-10%（高光/UI/特效）。遮罩管"禁什么"，和谐管"取什么"，60-30-10 管"各占多少"——三者合起来才是完整的色彩配置。

### 5.5 Albers 色彩相对性

Josef Albers 在《Interaction of Color》中证明了一个事实：**同一个颜色在不同背景下看起来完全不同。** 一个墨绿在白底上显得深、在红底上偏蓝、在同色底上几乎消失。大脑不是绝对色彩计，而是相对色彩比较器。

![Albers实验色板](../images/练手/练手05-fig05-Albers实验.svg)

*图 练手05.5：同一个色在不同底色上看起来截然不同。*

对游戏开发的直接影响：**你不能孤立地选颜色，必须在它最终出现的背景上测试。** 一个在灰色调色板里完美的血条红，放到绿色草地上可能完全不对——红绿并置让两色都更鲜艳。这也是练手04"先建明度骨架"的延伸：骨架是相对关系，色彩也是相对关系。L2 有 6 个屏幕实验让你亲身体验。

> **程序员类比 #6：** Albers 相对性 = 变量值依赖上下文。一个颜色的表现不是它的绝对 hex 值，而是 `color - neighbor_color` 的差值。孤立选色 = 不看作用域直接定义常量；在背景上测试 = 在真实调用点验证语义。

### 5.6 经典调色板 + 色彩脚本 + 词汇表

无需从零配色。社区有经过验证的经典调色板，Aseprite 直接导入：

![经典调色板陈列](../images/练手/练手05-fig10-经典调色板陈列.svg)

*图 练手05.10：五套经典像素调色板。更多见 lospec.com/palette-list。*

- **PICO-8（16 色）**：复古游戏标准色板。
- **DB16 / DawnBringer 16（16 色）**：极受推崇的 16 色板，色彩平衡极好。
- **Endesga 32（32 色）**：适用性极广，适合大多数项目。
- **Sweetie 16（16 色）**：明亮柔和，适合可爱风格。
- **ARQ4（4 色）**：极简 4 色板，适合限制练习。

**调色板管理（Aseprite）。** 创建自定义：选色 → 点前景色旁红色感叹号添加 → Options → Save Palette（`.ase`）。从图像提取：打开精灵 → Create Palette from Current Sprite。限制练习：16 色（宽松）/ 8 色（需规划）/ 2 色（极端，靠抖动）。

**调色板替换（Palette Swap）。** 只换调色板里的颜色，同一精灵变成不同角色（敌人变体），或同一场景呈现白天/黄昏/夜晚。一套素材，多套皮肤——游戏开发里极高效的技巧。

> **程序员类比 #7：** 经典调色板 = 社区维护的 `enum` 常量库，导入即用；调色板替换 = 运行时换 `enum` 实例——精灵索引不变，只换索引指向的颜色值，就像换主题配置而 UI 结构不动。

**gamineai 90 分钟 5 色脚本。** 在 90 分钟内用 5 个颜色锁定游戏色彩身份：①**暗色**——最深色调，阴影/暗部/夜晚基调；②**中暖色**——地面/皮肤/木质；③**中冷色**——天空/水面/补光；④**饱和强调色**——交互元素/主角标识/特效；⑤**高亮色**——高光/UI 文字/光源指示。工作流：选 5 色（10 分钟）→ 画 3 个关键帧（30 分钟）→ 检查色彩弧线（20 分钟）→ 扩展为完整色板（30 分钟）。

![色彩脚本5色模板](../images/练手/练手05-fig06-色彩脚本模板.svg)

*图 练手05.6：gamineai 5 色脚本模板。*

**色彩词汇表。** Vodeo Games《Beast Breaker》：黄色="安全/归属"、白色="主角，最高明度永远从背景跳出"、洋红色="腐败/危险/异类"。Portal：橙=玩家传送门 / 蓝=系统传送门。给每个核心色写一句功能定义——"这个色在游戏里表示______"——并在整个开发中坚持。当有人说"加个紫色吧"，你检查：紫色在词汇表里代表什么？已有语义就不能被用于纯装饰。

### 5.7 色盲无障碍

约 8% 男性有色觉缺陷（最常见红绿色盲）。三条原则：

1. **不纯靠颜色区分。** 敌/友不只靠红/绿，加不同形状（圆 vs 方）、不同图标、不同动画频率。
2. **明度对比兜底。** 关键 UI 去色检查——去色后还能区分吗？练手04 去色检验法的又一个应用。
3. **形状/图标双重编码。** "火=红、冰=蓝"不够——加火焰图标和雪花图标。颜色是主信号，图标是兜底。

> **程序员类比 #8：** 关键信息不编码进单一通道。颜色是一个通道，明度是另一个，形状/图标是第三个——冗余编码到多个独立通道，单通道失败时其他通道兜底。最简检验：UI 截图去色，信息还在吗？

### 5.8 练习

> **贯穿式角色项目（续）：** L1 给 `character-v03-value.aseprite` 穿上色彩皮肤——选有限调色板、用东式色相偏移着色，存为 `character-v04-color.aseprite`，制作02 收口。

#### L1 · 给贯穿角色上色（60-120 分钟）

**步骤：** ①定色彩词汇表：列出角色 5 种信息类型，每种写一句"这个色表示______"；②选调色板：从经典色板选一套删到 8 色以内，或用 gamineai 5 色脚本；③定和谐方案 + 60-30-10 比例；④灰度→上色：复制 v03，新建 Overlay/Color 图层叠加颜色，只改色相饱和度，不动明度结构；⑤色相偏移着色：高光向暖偏 5-10°、阴影向冷偏 5-15°、V 降 15-25%；⑥阴影不纯黑；⑦Albers 检查——把角色放进假设的背景色上看看；⑧去色验证——Ctrl+Shift+U，焦点还在吗？保存为 `character-v04-color.aseprite`。

**合格标准：** 写了词汇表 + 和谐方案 + 比例 / 调色板 ≤ 8 色 / 至少一条 ramp 用东式色相偏移 / 阴影无纯黑 / 去色后焦点仍在 / 文件存为 `character-v04-color.aseprite`。

#### L2 · Albers 实验（15 分钟）

画三个 200×200 背景块（纯白、纯红 #E74C3C、墨绿 #2E6B3E），每块中心放一个 60×60 墨绿方块（完全相同色值）。对比三列：同一个墨绿，在白上最深、在红上偏蓝偏冷、在绿上几乎消失。再试变体：同一橙放灰底 vs 蓝底；同一蓝放黑底 vs 白底；红黄小方块交替排列观察橙色视错觉。

**合格标准：** 你亲口说出"同一个色在不同背景上确实不同"，并从此不再孤立选色。

#### L3 · 经典色板提取 + 移植 + 换肤（1-3 小时）

①从 Lospec 下载 Endesga 32 导入 Aseprite，分析配色结构；②用 Endesga 32 给 v04 换一套色（palette swap）；③只用 Endesga 32 的 4 色重画角色；④再换 Sweetie 16 / DB16 各试一次，对比三套色板给角色的不同感觉。

**合格标准：** 能说出三套色板的差异，并成功用至少两套给角色换肤且明度结构未受损。进阶思考：有 32 色角色会比 8 色版更"好"还是更"乱"？

#### 去色检验法（贯穿全章）

每加一个新色，Ctrl+Shift+U 去色检查一次。如果你发现彩色版去色后全都没了——新加的颜色明度和已有色撞了，回 5.4 查遮罩和和谐方案。

### 5.9 常见错误

**错误 1：色彩过多导致"脏"。** 画面灰蒙蒙，每个单色都挺好看但合起来不舒服。**修复**：回 gamut mask，选一个三角形，域外色全降饱和或删除——好画面的秘密往往不是加色，是删色。

**错误 2：交互信息被颜色淹没。** 玩家不知道点哪。**修复**：交互元素必须用画面中饱和度最高或对比最强的色。《蔚蓝》原理：冷紫环境 → 暖橙交互，永远相反。

**错误 3：忽略背景对颜色的影响。** 调色板里选好的色放到场景里完全不对。**修复**：永远在实际背景上测试。角色会出现在草地/雪地/沙漠三张图，就要在三张图上分别测（5.5 Albers）。

**错误 4：直接加黑做阴影。** 阴影区很脏、很死。**修复**：阴影公式 = V 降 20% + H 向蓝紫偏 10° + S 升 5%。阴影立刻有空气感（5.3）。

### 5.10 小结

色彩的首要身份是信号，其次才是装饰。调色板越大，每个色的力量越小。用功能分层排优先级；用 HSV 三轴选色；用东式色相偏移让 ramp 有空气感；用 gamut mask + 和谐方案 + 60-30-10 定取色/禁色/比例；用经典调色板和 5 色脚本拿来即用；用色彩词汇表给每个色赋予功能语义。记住 Albers：颜色不能孤立选择——它永远在和邻居对话。

**如果只记住一句话：** 先用练手04 的明度骨架打底（HTML），再套有限调色板 + 东式色相偏移（CSS）——明度是结构，色相偏移是给 ramp 的灰阶换成 HSL 空间曲线插值。

**上手行动：** 打开 `character-v03-value.aseprite`，选 Endesga 32 删到 8 色，用东式 hue-shifting 上色，存成 `character-v04-color.aseprite`。按一次 Ctrl+Shift+U——去色后角色比练手04 版更清晰、彩色比西式更有空气感，说明色彩直觉长了一层。制作02 收口。
