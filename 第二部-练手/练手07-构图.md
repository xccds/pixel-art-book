# 练手07 构图：观众的眼睛跟着你的设计走

### 7.0 这一章解决什么问题

练手06 你给贯穿角色穿上了质感皮肤——存成 `character-v05-texture.aseprite`，金属硬边、布料颗粒、皮肤棋盘格各就各位。可一旦你想把它放进一个场景——一棵树、一块石头、一片天空——新的困惑立刻冒出来：为什么你的角色单独看挺好，放进场景里就"找不到"了？为什么观众第一眼看的是背景里那块亮石头而不是你的主角？为什么《空洞骑士》每一帧截图都有一个明确的视觉重心，你的画面却像一张"所有东西都一样重要"的说明书？

这一章解决的就是**从"画好单个元素"到"组织整个画面"的跨越**——在质感之上，怎么用构图控制观众的眼睛先看哪、再看哪、停在哪。

观察02 把"构图"列为八种数据类型之一，给了你一句话定义：元素在画面中的组织方式——焦点、平衡、视线流。这一章把这句话彻底拆开：构图为什么是"手动布局引擎"、视觉权重怎么算、引导线怎么铺、动作线怎么定。

书 A 给了布局引擎、视觉权重五规则、引导线=超链接、三分法=CSS grid、取景=overflow:hidden、三鲁棒性、三游戏类型构图策略这套框架。书 B 的构图间奏章补了古典规则、负空间、视觉权重与平衡、三引导工具（明度/饱和度/细节密度）、动作线（C/S/Z/竖/斜）、场景分析、构图清单。**关键合并**：A 的"视觉权重五规则"和 B 的"视觉权重与平衡"是同一个权重体系的两半，本章统一成一套（五规则算权重 + 对称/不对称排权重）；A 的"引导线"和 B 的"三引导工具"是两种不同的视线引导机制（线条引导 vs 对比引导），本章统一成一个"视线引导"框架，不重复讲两遍。

> **贯穿式角色项目（续）：** 练手01 你画了 `character-v01-outline.aseprite`；练手02 长出体积存成 `character-v02-volume.aseprite`；练手04 重打光存成 `character-v03-value.aseprite`；练手05 上色存成 `character-v04-color.aseprite`；练手06 加质感存成 `character-v05-texture.aseprite`。**本章 L1 给这个角色定动作线 + 构图场景**——先用一条动作线（C/S/Z/竖/斜）重新摆姿态，再把它放进一个简单场景（一棵树 + 一座远处建筑），用三分法落位、引导线指向、视觉权重排序组织画面，存成 `character-v06-pose.aseprite`。它将在**制作02（像素角色工作流）**收口成可导入引擎的完整角色。所以 L1 你画的不只是练习，是给你贯穿角色一个"有构图意识的登场"。

你只需要 Aseprite（纸笔也行），加一双愿意"先设计视线路径再动笔"的眼睛。

### 7.1 程序员视角：构图到底是什么

#### 7.1.1 构图的"布局引擎"本质

如果你写过前端，你应该知道一个页面的视觉结果是由布局引擎（Layout Engine）计算出来的——浏览器解析 CSS，计算每个元素的盒模型大小和位置，然后根据 `flex`/`grid`/`float` 规则把元素放到对应的像素坐标上。这个"计算位置"的过程就是构图。

但**你不是浏览器。你没有一个会自动算位置的渲染引擎。**你画图的方式是直觉性的——你看到一个画面在脑子里，然后把元素摆出来。问题是：直觉摆出来的东西，经常缺一个清晰的焦点——就像你把所有 div 都设成 `flex: 1 1 auto`，结果谁也主导不了谁。

构图就是手动扮演布局引擎。你需要在脑子里提前算好三个参数：

1. **焦点（Focal Point）**——画面中视觉权重最高的元素。观众第一眼看这里。一个画面必须有且只有一个焦点的优先级高于所有其他元素。如果一个画面有五个"看起来都差不多重要"的元素，那就等于没有焦点。
2. **视线路径（Visual Path）**——观众的眼睛从焦点出发，按照什么顺序浏览画面的其他部分。Z 型路径（从左到右、从上到下，像读书一样）是最自然的路径。引导线可以改变这个默认路径。
3. **次要元素分布**——焦点的视觉"同伴"，它们的存在必须服务于一个目的：要么引导视线回到焦点，要么提供对比衬托焦点，要么填充空间但不能抢焦点。

| 构图概念 | CSS 等价 | 作用 |
|---|---|---|
| 焦点 | `order: -1` / `z-index: 999` | 第一个被眼睛看到的元素 |
| 视觉权重 | `flex-grow: N` | 元素在视觉上占据的"分量" |
| 引导线 | `<a href="#target">` | 视线跳转到指定位置 |
| 三分法 | `grid-template: 1fr 1fr 1fr` | 划分画面空间的结构框架 |
| 平衡 | `justify-content: space-between` | 元素在画面左右的分布 |

*图 练手07.1：八个构图工具形成一个工具箱。实际使用时它们不是独立的——一个画面通常同时用到 3-4 种工具。*

#### 7.1.2 视觉权重与平衡——五规则 + 两补 + 对称/不对称

视觉权重不是主观感觉。它是可以被拆解成规则的。书 A 给了五条核心规则，书 B 补充了两条因素和平衡原理——本章把它们统一成一套完整的权重体系。

**五条核心规则（算权重）：**

**① 亮度规则。** 在暗背景上，亮的物体权重最大。在亮背景上，暗的物体权重最大。这是一个**对比效应**——不是绝对值，是**相对值**。一片全亮画面中那个唯一的暗点比太阳还显眼。

**② 饱和度规则。** 高饱和度的暖色（红、橙、黄）比低饱和度的冷色（灰蓝、灰绿）更吸引注意。红色是视觉系统中最强的注意力捕获色之一——人类的视觉皮层对红色-绿色对比通道（Red-Green Opponent Channel）的反应最强烈 [2]。

**③ 尺寸规则。** 大比小权重高。但这个规则和对比度规则会相互抵消或放大——一个巨大的灰色圆形可能比一个小的红色圆点的视觉权重还低，取决于灰色多灰、红色多红。

**④ 人脸优先规则。** 人脸——尤其是眼睛——是视觉系统自动检测的最高优先级目标。人脑中有专门的**梭状回面孔区**（FFA, Fusiform Face Area）负责面孔检测，它的处理速度是 100ms 级别的预注意处理（Preattentive Processing）[3]。所以一个画面中只要有面部——哪怕是远处的一个小脸——它就会抢夺焦点。这个特性在游戏构图中极其重要：如果你想表达"这个角色是画面焦点"，不需要画得超级大——只要把它放在正确的位置并在面部加一些细节就够了。

**⑤ 复杂度规则。** 细节密集的区域比平坦的区域更吸引注意。纹理丰富的石头会比光滑的空白墙壁先被看到——眼睛天然会被"信息密度高"的地方吸引。

**两条补充因素（书 B 增补）：**

**⑥ 独立性。** 一个孤立的元素比挤在一堆同类中的元素权重高——单独一粒棋子比棋盘中任意一粒更显眼。这就是负空间的作用（7.1.5 会展开）。

**⑦ 边缘位置 = 力矩。** 画面边缘的元素比靠近中心的元素拥有更长的"视觉力矩"——就像跷跷板上离支点远的一端用更小的重量就能平衡更大的近端重量。一个小物体放在画面极边缘，可以平衡中心区的一个大物体。

![视觉权重分布](images/练手/练手07-fig02-视觉权重分布.svg)

*图 练手07.2：视觉权重不是绝对值——是相对值。红色的权重 5 圆点吃掉最多的视觉注意力，灰色小圆最后才被注意到。*

**视觉权重的加减法：** 你不一定能直接增大焦点元素的权重——也许它已经画好不能改了。但你总可以**减小周围元素的权重**。把背景变暗、把次要物体的对比度降低、把不需要的元素模糊处理——这些减法操作和加法一样有效。

**平衡：权重的排列方式。** 算完每个元素的权重分数，接下来是排列。书 B 给了两种平衡模式：

- **对称平衡**：左右元素重量相同，安静、稳定、庄严。适合建筑、徽章、Boss 特写。
- **不对称平衡**：左侧一个"重"元素 + 右侧多个"轻"元素来平衡（靠边缘位置力矩补重量差）。适合动作场景、故事性画面。实用方法：想象画面中心有一个支点，左侧大角色（重），右侧放一个小道具+亮色高光（轻但靠外）来平衡。

> **程序员类比 #1：** 不对称平衡 ≈ CSS `justify-content: space-between`——左边一个大块，右边一个小块推到边缘，中间留白。视觉上平衡了，但比对称平衡更有张力。想象画面中心有一个支点（`align-items: center`），你用 `flex-grow` 大的和 `margin-left: auto` 推远的小的来配平。

#### 7.1.3 视线引导——引导线 + 三引导工具的统一

构图控制观众眼睛靠两种机制。书 A 讲的"引导线"和书 B 讲的"三引导工具"不是重复——它们是两种不同的视线引导机制，本章统一成一个框架：

**机制一：对比引导（书 B 的三引导工具）——"看最亮/最鲜艳/最细节的地方"。** 这是视觉权重的**主动应用**：你故意让焦点的某个维度对比最强，眼睛被对比差吸引过去。

- **明度对比引导**：确保画面中最高对比度区域正好是你希望观众最先看到的地方。想让人看角色头部？头部颜色与背景的明度差要最大。
- **饱和度对比引导**：高饱和度区域比低饱和度区域更吸引眼球。前景角色高饱和、背景降饱和。
- **细节密度引导**：细节多的区域吸引注意力，细节少的区域让眼睛休息。角色面部增加细节，背景减少细节。在像素画中，"细节密度"指的是像素变化的频率（颜色/明度变化的密集程度）。

三种工具组合使用最强：**高对比度 + 高饱和度 + 高细节密度**同时集中在一个区域，这个区域就是无可争议的视觉焦点。

**机制二：线条引导（书 A 的引导线）——"沿着这条线看过去"。** 这不是靠对比吸引，而是靠**物理方向**引导。你打开一个网页，URL 里有 `<a href="#content">跳到正文</a>`。你一点，页面就跳过去了。引导线就是画面上的链接——它不是用来点的，是用来"看"的。

> **程序员类比 #2：** 引导线 ≈ 视线 hyperlink 锚点。道路指向城堡 = `<a href="#castle">`。栏杆指向角色 = `<a href="#hero">`。光线从窗口打到宝箱 = `<a href="#loot">`。构图就是你在视觉上铺设多个锚点链接，指挥观众的眼睛一个接一个地点击下去。

有效的引导线形状：直线/斜线（道路、河流、铁轨——人眼处理最快的方向 [4]）、曲线（弯曲小路——更有探索感）、隐含线（角色目光方向、手指指向——即使无实际线条眼睛也本能跟随 [3]）、边缘线（建筑轮廓、地平线——形成视觉"通道"）。

**陷阱：** 引导线必须指向画面内的某个东西。如果一条路指向画框右边缘的外面，而边缘外面什么都没有——那观众的视线就被"截断"了。一旦视线离开画面，它大概率不会再回来。这不是引导，这是放逐。

![引导线](images/练手/练手07-fig04-引导线.svg)

*图 练手07.4：三种典型的引导线——道路指向远景、栏杆指向角色、光线指向宝箱。每种引导线的来源不同，但它们都指向画面内的一个具体锚点。*

**两种机制的分工：** 对比引导决定"观众第一眼看哪"（权重排序），线条引导决定"观众接下来沿着什么路径看"（流向设计）。一个画面通常先用对比引导锁定焦点，再用引导线设计从焦点出发的视线路径。

#### 7.1.4 古典构图法则速览——三分/对角/取景/黄金比

构图有几条从古典绘画传下来的法则。书 B 列了五条，其中三分法和引导线已在上面展开（引导线在 7.1.3，三分法在 7.2.2 细化），这里速览其余三条：

**对角线构图。** 关键元素沿对角线方向排列，创造动感和延伸感。左下到右上 = 正向、积极、上升；左上到右下 = 舒缓、下降、稳定。适合战斗场景、角色冲刺。在 32×32 像素画中，如果角色正在挥剑，让剑的轨迹从左上到右下形成一条斜线，整个画面立刻充满动感。

**取景（Framing）。** 用前景元素（树枝、拱门、窗口、门框）形成一个天然框架，把视线"框"在里面的主体上。逻辑是**限制视线的横向逃逸**——就像 `overflow: hidden`，超出边界的内容被裁剪，不看。最经典的例子是《塞尔达传说：旷野之息》里林克站在山洞门口望向远方——山洞洞口就是天然框景。

**黄金比。** 约 1:1.618 的比例。主体占据面积与留白面积的比例接近黄金比时，视觉上最和谐。在像素画实践中不必精确计算——记住简化原则：**主体不要太小也不要太大，通常占画面的 1/3 到 1/2**。

**负空间。** 画面中未被主体占用的"空白"区域。负空间的形状应该**有意设计**，而不是"剩下的地方"。在移动方向/视线方向留出更多空间——角色面向右侧，左侧空间应小于右侧。负空间在构图中的三个功能：（1）给眼睛休息区；（2）通过对比突出正形；（3）建立视觉节奏。完整展开见练手03。

#### 7.1.6 动作线——角色姿态的主干向量

这是书 B 的核心贡献，书 A 完全没覆盖。动作线（Line of Action）是贯穿角色身体的一条看不见的曲线或折线，传达角色的动作趋势和能量。

**五种动作线形状：**

- **C 形曲线**：灵活、有机、柔软。适合柔和前倾、女性角色。
- **S 形曲线**：极度灵活、优雅。适合站姿、舞蹈动作。
- **Z 形折线**：力量、冲击、硬朗。适合攻击动作、机械角色。
- **垂直直线**：静止、稳定、庄严。适合守卫、雕像。
- **倾斜直线**：动势、不稳定。适合奔跑、即将跌倒。

![动作线形状指南](images/练手/练手07-fig06-动作线形状指南.svg)

*图 练手07.6：五种动作线形状叠加在简化角色姿态上。先定线再画人——动作线是姿态的 signature，细节是 implementation。*

> **程序员类比 #3（动作线核心类比）：** 动作线 = **角色姿态的主干向量，像函数的主返回值**——细节是副作用（side effects），动作线是函数签名（function signature）。先定动作线再画细节 = 先写函数签名再填实现。你不会先写 50 行函数体再回头猜它该返回什么类型——同理，你不会先画盔甲纹理再回头想这个角色在做什么动作。**动作线错了，细节再精致也救不回来——就像函数签名错了，实现再优雅也是死代码。**

**动作线的像素画应用：** 在角色草稿阶段，不要从画眼睛或盔甲开始——先用 1 像素宽的线条画出角色的动作线，然后沿着动作线添加几何体（球=头、椭圆柱=躯干+四肢），最后在几何体基础上添加细节。

**错误示范与落地感：** 先画了一个正面直立的分腿站姿，然后想让它显得"在奔跑"——不可能。因为你已经用垂直动作线锁死了角色。动作线决定了角色能够传达多少"生命力"。动作线之外，角色的体重要有"落地感"——重心应该在支撑面（双脚之间的区域）上方。正在出拳的角色重心在靠前的脚上；正在后撤的角色重心在后面的脚上。

### 7.2 理论最小集

上一节列了构图工具和动作线。但如果你只挑**能带来最大边际收益的**，它们是：视觉权重清单、三分法落位、引导线设计、动作线设计。另有一个独立开发者必须知道的特殊话题——游戏构图的鲁棒性，和三游戏类型的构图策略差异。

#### 7.2.1 工具一：视觉权重清单（画前检查）

在你画任何一个画面之前，先在你的草图旁边列一个清单：

1. 这个画面里什么东西最大？（尺寸权重）
2. 什么东西最亮/最暗？（亮度权重）
3. 什么东西饱和度最高？（色彩权重）
4. 有没有人脸？（人脸权重——最高优先级）
5. 有没有高对比度区域？（对比度权重）

**然后排序。** 权重最高的东西是焦点。如果焦点不是你本来想表达的东西——改。降低它的对比度、缩小它、把它挪到画面边缘——总之让它闭嘴，让真正的主角说话。

#### 7.2.2 工具二：三分法落位（先画线再画物体）

把画面在水平和垂直方向各切两刀，分成 3×3 网格。四个交点——不是画面正中心——是自然视觉中**最舒适的主体位置**。

![三分法](images/练手/练手07-fig03-三分法.svg)

*图 练手07.3：三分法把画面切成 3×3。四个交点是最佳的主体位置——因为人眼的自然 Z 型扫描路径在路过交点时停留时间最长。*

为什么正中心反而不好？这不是什么玄学——中心对称是**最静态**的构图方式。把主体放正中央，画面就失去了方向感——左右完全对称，视线没有理由往任何方向移动。画面就像一张被锚定的船：不左不右，不偏不倚。这适合拍证件照，不适合讲故事。但这不是说永远不用中心构图——《塞尔达传说》的镜头经常让林克居中——因为俯视角游戏需要中心作为稳定锚点。

在 64×64 的像素画布上，四个交点约在 (21,21)、(21,43)、(43,21)、(43,43) 像素位置。交点不是精确坐标——是区域，交点周围有一个约 1/8 画面宽度的"舒适区"。

#### 7.2.3 工具三：引导线设计（至少一条指向焦点）

如果你只加一条引导线，选最有力的那种：**一条路、一条栏杆或一束光，直接指向你的焦点。** 一条引导线足以把你画面的视线管理能力从"随机"提升到"被设计"。加上第二条引导线，你可以创造**三角关系**（Triangular Composition）——两条引导线从不同方向汇聚到同一个焦点。这是古典油画中最常用的构图结构，独立游戏里一样有效。两条引导线交叉，焦点就获得了"双倍视觉拉力"。

#### 7.2.4 工具四：动作线设计（先定线再画人）

在画角色之前，先决定它的动作线形状。问自己：这个角色在做什么？它传达的能量是什么？

- 站着警戒 → 垂直直线
- 奔跑冲刺 → 倾斜直线
- 攻击挥拳 → Z 形折线
- 优雅站立 → S 形曲线
- 柔和前倾 → C 形曲线

用 1 像素宽的线条在草稿层画出动作线，然后沿着它搭建几何体，最后加细节。**动作线是第一笔，不是最后一笔。**

#### 7.2.5 游戏构图的鲁棒性——三要求

这是本章最重要但最被忽视的内容。静态画（油画、插画、海报）的构图可以精确到像素——因为你知道观众在哪看、从什么距离看、画面多大。但游戏构图必须面对三个不确定：

**① 焦点可能移动。** 主角在屏幕中间跑、跳跃、攻击——构图焦点不停改变。解法：不是把所有东西画死，而是**始终保持一个高对比度的视觉锚点**（主角轮廓、血条红色、武器亮光），锚点跟着主角移动。同时确保环境元素不形成比主角更强的新焦点。

**② UI 会遮挡。** HUD（血条、小地图、道具栏）压在画面上层——不参与构图又无法忽略。解法：**构图时把 UI 考虑进去。**在草稿里画出 UI 占位的灰色方块，只在非 UI 区域里做构图。有效的游戏构图是画面空间减去 UI 占用后的剩余区域。

**③ 分辨率和窗口变化。** 玩家屏幕可能是 1080p、1440p、720p，可能全屏或窗口化。解法：不要把关键信息放在画面绝对边缘。给焦点留出至少 10% 的画面边距——无论画面怎么裁，焦点仍在安全区内。

这三个要求合在一起叫构图的**鲁棒性**——即使在环境变化的条件下，你的构图策略仍然有效。这是游戏构图区别于传统构图的本质特征。

#### 7.2.6 三种游戏类型的构图策略差异

不同游戏类型需要完全不同的构图思维。一种游戏的解决方案可能是另一种游戏的构图毒药。

![游戏类型构图](images/练手/练手07-fig05-游戏类型构图.svg)

*图 练手07.5：三种游戏类型的构图策略完全不同。平台跳跃是横向扫读，俯视角是全局监控，第一人称是深度探进。*

**① 平台跳跃（横向视线流，Z 型扫描）。** 经典：《超级马里奥》《蔚蓝》《空洞骑士》。相机向右跟随——视线路径左上→右上→左下→右下（Z 型）。策略：把关键信息放在 Z 路径上——左上放 HUD、右中放前方方向（前方留出视线空间）、右下放玩家角色。

**② 俯视角（全局均衡，中心锚定）。** 经典：《星露谷物语》《暗黑破坏神》《以撒的结合》。相机俯视——只有 360° 包围空间。视线从中心向四周辐射再回收。策略：中心恒定为最高视觉权重，四周信息均衡，环境标志物诱导方向。

**③ 第一人称（深度引导，Z 轴纵深）。** 经典：《守望先锋》《生化奇兵》《半衰期》。相机即眼睛——焦点由深度决定。策略：远处焦点放在画面消失点附近，Z 轴中间放置引导元素牵引视线从近到远，近处元素只用于参考框架。

#### 7.2.7 场景分析——战斗 vs 平静

同样的构图工具，用在不同的情绪场景里选择完全不同。**战斗场景**选对角线（角色和敌人分居左下/右上形成冲突动态线）+ 引导线（剑尖指向敌人）+ 明度引导（脸和剑是画面中最亮的部分）+ 不对称平衡（左侧大角色 vs 右侧边缘小敌人靠力矩配平）——整体高对比、有冲突感。**平静场景**选三分法（地平线在下 1/3，角色在左下交点）+ 对称平衡 + 低对比度（全画明度范围窄）+ 大量留白——整体宁静空旷。同一个工具箱，**构图没有"对的"构图，只有"对当前叙事意图有效的"构图。**

#### 7.2.8 构图清单

画完一个场景后，用这张清单自查（书 A 的清单 + 书 B 的清单合并）：

1. □ 观众的视线第一时间落在哪里？是否是我想要他们看的地方？
2. □ 是否使用了明度/饱和度/细节密度来引导视线？（对比引导）
3. □ 是否有至少一条引导线指向焦点？（线条引导）
4. □ 主角的面向/移动方向是否有足够的负空间？
5. □ 画面左右是否平衡（对称或不对称）？
6. □ 角色的动作线传达了我想要的姿态吗？
7. □ 前景、中景、背景是否有层次？
8. □ 去掉颜色看灰度版本——构图还成立吗？

### 7.3 练习

本章练习逐级递进，每级有明确"合格标准"。三个练习建议分三天。

> **贯穿式角色项目（续）：** 练手06 你存了 `character-v05-texture.aseprite`（质感皮肤完成、明度骨架未塌）。**本章 L1 给这个角色定动作线 + 构图场景**——先用一条动作线重新摆姿态，再把它放进一个简单场景，用三分法+引导线+视觉权重组织画面，存成 `character-v06-pose.aseprite`。这个角色将在**制作02（像素角色工作流）**收口成可导入引擎的完整角色。所以 L1 你画的不只是练习，是给你贯穿角色一个"有构图意识的登场"。

#### L1 · 给贯穿角色定动作线 + 构图场景（约 60-120 分钟）

**目标**：把练手06 的 `character-v05-texture.aseprite` 从"直立站着"升级为"有动作线的姿态 + 有构图的场景"——先定动作线再摆姿态，再用三分法+引导线组织画面。

**你需要**：练手06 的 `character-v05-texture.aseprite`、Aseprite。

**步骤**：
1. **选动作线**：在纸上画出五种动作线（C/S/Z/竖/斜），选一条最适合你角色性格的——守卫型选竖线、攻击型选 Z 线、探索型选倾斜线。在 Aseprite 新建图层，用 1 像素宽的线条画出这条动作线。
2. **沿动作线摆姿态**：复制 `character-v05-texture.aseprite` 为 `character-v06-pose.aseprite`。沿着动作线调整角色姿态——把直立站姿改成符合动作线能量趋势的姿势。重心要在支撑面上方。
3. **加场景三元素**：在角色周围加一棵树 + 一座远处建筑 + 一片天空（总共三个环境元素）。限制数量保证构图清晰。
4. **三分法落位**：把角色放在一个三分线交点上（不要放正中心），树和建筑放在其他交点或作为引导线元素。
5. **引导线设计**：至少加一条引导线指向角色——可以是地面的路径、栏杆、或光线。
6. **视觉权重排序**：检查清单——角色是不是最大/最亮/最饱和/有人脸？如果不是，降低环境元素的对比度和饱和度让角色"说话"。
7. **去色验证**：按 Ctrl+Shift+U——去色后焦点还在角色身上吗？构图还成立吗？
8. 保存为 `character-v06-pose.aseprite`。

**合格标准**：
- [ ] 纸上画了五种动作线并选了一种（能说出为什么选这条）。
- [ ] 角色姿态沿动作线摆放（不是直立站姿），重心在支撑面上方。
- [ ] 场景有 3 个环境元素，角色在三分线交点上。
- [ ] 至少一条引导线指向角色。
- [ ] 视觉权重排序后角色是第一焦点（去色验证通过）。
- [ ] 文件存为 `character-v06-pose.aseprite`，准备交给制作02 收口。

#### L2 · 黑白灰构图——控制视线路径（约 15-30 分钟，纸笔）

**目标**：只用黑白灰（铅笔或黑色水笔），画一个有明确视觉重心的简单室内场景，并标注预期视线路径。

**步骤**：
1. 准备一张白纸和一支铅笔。画一个简单的室内场景——一张桌子、一把椅子、一扇窗户、一扇门、一个人（可以是火柴人）。限制在 5 个元素以内。
2. 确定你的焦点——你希望观众第一个看到什么？用最重（最深色）的线把它画出来。
3. 用箭头标记你认为观众会看到东西的顺序：①→②→③。画箭头时想象你是一个第一次看到这张图的观众——诚实，不要自欺。
4. 找一个人（不告诉他你的焦点是什么），给他看 3 秒后盖住画面，问他"你第一个看到的是什么？"
5. 如果他说的和你的焦点一致 → 你的构图有效。不一致 → 去看你的视觉权重分配——什么东西比你的焦点更亮/更大/更有对比？

**合格标准**：
- [ ] 画面有一个可辨识的焦点（不只是"整张图都是焦点"）。
- [ ] 你标注了明确的视线路径（至少 3 步：焦点 → 次要元素 A → 次要元素 B）。
- [ ] 测试对象第一个看到的东西与你预设的焦点一致。
- [ ] 画面中没有元素把引导线指向画框外。
- [ ] 只用了黑白灰——没有颜色帮助，纯靠明度和构图控制视线。

#### L3 · 同一场景的 3 个构图变体（约 1-3 小时，纸笔或数位板）

**目标**：画同一场景的三个构图变体，每个构图传达不同的情绪。只改变构图——场景内容不变。

**步骤**：
1. 选定一个简单的场景内容：一个角色 + 一棵树 + 一座远处的建筑。总共三个元素——限制数量以保证对比清晰。
2. 画三个变体（每个变体可以用一张 A5 纸或数位板上独立的图层）：
   - **变体 A · 远景（渺小）**：角色占画面不到 10%。树和建筑在远处。传达"渺小、孤独、被环境吞噬"。用三分法+大量负空间。
   - **变体 B · 中景（叙事）**：角色约占 20-30%。角色和树在同一层，建筑在远处。传达"观察、叙事、有事即将发生"。用三分法+引导线。
   - **变体 C · 近景（紧张）**：角色占据画面 40-60%。脸或上半身充满画面，树和建筑在边缘。传达"紧张、亲密、内心情绪"。用对角线+高对比+不对称平衡。
3. 对每个变体，标注你用的构图工具（三分法落位？引导线？框景？对比？）和动作线形状。
4. 把三张变体并排放在一起，观察：同一个场景的三个构图是否真的传达了三种不同情绪？

**合格标准**：
- [ ] 三个变体的场景元素相同（一个角色 + 一棵树 + 一座远处建筑），仅构图改变。
- [ ] 变体 A 的角色确实很小，画面感觉空旷。
- [ ] 变体 B 的角色适中，能辨识出叙事感。
- [ ] 变体 C 的角色占据大部分画面，有逼近感。
- [ ] 每个变体标注了使用的构图工具和动作线形状。
- [ ] 三张放在一起，能明显感受到情绪在"渺小"→"观察"→"紧张"之间梯度变化。

### 7.4 常见错误与诊断

构图是你已经画了很多年、但从未被正式纠正过的技能。以下是你画的每张图里可能正在犯的错误。

**错误 1：焦点模糊 → 没有明确视觉重心。** 画面看起来"还行"，但观众说不出哪个是主角、哪个是背景。元素之间的视觉权重差不多——角色和旁边的树一样重。**原因**：你花了同样的时间和心力去画每一个元素——投入被"平均化"了。这在游戏开发中尤其常见——美术需要画一整张"完整的背景"，于是你下意识把每个角落都画得同样详细。**修复**：确定**唯一的焦点**。先画它，花 50% 的时间在上面。焦点搞定以后，其余元素用**减半处理**。把画面缩放到 10% 大小（眯眼也行），只看谁能抢到第一个注意力——那是你的焦点。不是你想的那个就改。

**错误 2：视线被截断 → 引导线指向画面外。** 一条路、一个栏杆或角色的目光方向指向画面边缘——观众的眼睛跟着它走，走到边缘后突然无路可走。视线"滑出"了画面，很难再回来。**原因**：你不自觉地把引导线画到了边缘——画的时候只关注了画布内部，忘记检查"这条线指向了哪里"。**修复**：检查每一条明显的引导线是否指向了画面**内部**的某个具体元素。如果某条线指向边缘，把它弯折——让它在碰到边缘前转向，指向焦点。或者，在引导线的尽头放一个"缓冲器"——一棵树、一个路标、一个发亮的小建筑物——让视线被接住而不是滑走。

**错误 3：信息过载 → 视觉拥堵。** 画面塞得太满——每个角落都有东西，每个东西都有细节。观众不知道往哪看，因为无论往哪看，都有一堆东西"在晃"。**原因**："多就是好"的默认思维。**修复**：给画面留"空白区"。空白不一定是白色——可以是任何低纹理、低对比度的区域（一片均匀的天空、一片水、一面简单的墙）。空白区是你的**视觉休息区**。用 **vignette** 技巧——让画面四角压暗或边缘模糊——让观众注意力自然集中在中央。绝大多数游戏在过场时都会这样处理。

**错误 4：无法兼容 UI → 焦点被遮挡。** 你构图时画了一个漂亮的焦点——但当你把 UI（血条、小地图、道具栏）叠上去之后，焦点完全被盖住了。**原因**：UI 是完全在构图之外的"叠加层"，但它的视觉权重是真实的——HUD 元素通常颜色鲜明、对比度高。**修复**：在画草图时就把 UI 占位区域用灰色半透明矩形画出来。只在**非 UI 区域**里设计焦点和引导线。让 UI 风格在视觉上与游戏画面形成"层差"——UI 用硬边、高对比、几何化处理，视觉系统会把它们识别为"界面"和"世界"分开。

**错误 5：动作线锁死 → 姿态没有生命力。** 先画了一个正面直立站姿，然后想让它显得"在奔跑"——不可能。因为你已经用垂直动作线锁死了角色。**原因**：动作线是最后一笔而不是第一笔——先画了细节再想姿态。**修复**：动作线是第一笔。先定动作线形状（奔跑=倾斜线、攻击=Z 线），再沿动作线搭建几何体，最后加细节。**动作线错了 = 函数签名错了 = 实现再优雅也是死代码。**

**错误 6：只使用一种构图工具 → 画面单调。** 你学会了三分法，于是你每张图都把主角放在右上交点。太规矩，太可预测。即使工具本身好，反复用同一个也会让画面失去生命感。**原因**：把工具当法则。**修复**：偶尔**刻意违反三分法**——主体放正中央、极左边缘、极上边缘——看看传达的是什么情绪。练习组合工具——三分法 + 框景 + 一条引导线——三种工具合力，比单个工具的力量大得多。

### 7.5 小结

构图是**视觉信息流设计**。你不是在"画画"，你是在决定观众的注意力分配——什么看第一、什么看第二、什么只看一眼、什么直接被忽略。这是一张画面的"阅读指南"。

**记住六件事**：
1. **构图 = 手动布局引擎**——你没有一个会自动算位置的渲染引擎，你手动算焦点、视线路径、次要元素分布。
2. **视觉权重五规则 + 两补 + 平衡**——亮度/饱和度/尺寸/人脸/复杂度算权重，独立性/边缘力矩是补充，对称/不对称是排列方式。视觉权重近似 CSS `flex-grow`：权重大的占据更多视觉空间。
3. **视线引导两种机制**——对比引导（明度/饱和度/细节密度）决定"第一眼看哪"，线条引导（引导线=hyperlink）决定"接下来沿什么路径看"。
4. **动作线 = 函数签名**——先定动作线再画细节 = 先写函数签名再填实现。动作线错了，细节再精致也是死代码。
5. **游戏构图必须鲁棒**——因为焦点会移动、UI 会遮挡、分辨率会变化。把 UI 考虑进构图，给焦点留出 10% 安全边距，确保你的构图在不同屏幕上都有效。
6. **三种游戏类型各有一种视线模式**——平台跳跃=Z 型扫描（左上到右下）、俯视角=中心辐射（360° 扫描再回到中心）、第一人称=Z 轴纵深（视线被透视推向消失点）。

**如果只记住一句话：** 构图就是你手动扮演布局引擎——用视觉权重算分量，用引导线铺链接，用动作线定姿态签名，用鲁棒性保证不同屏幕上都有效。

**上手行动**：今晚打开练手06 的 `character-v05-texture.aseprite`，按 L1 给它定一条动作线、摆一个有能量的姿态、放进一个三分法构图的场景，存成 `character-v06-pose.aseprite`。完成后去色——如果焦点还在角色身上、动作线一眼可读、环境元素没有抢戏，你的构图直觉就长了一层。这个角色将在制作02 收口。

### 7.6 扩展阅读

1. **《Framed Ink: Drawing and Composition for Visual Storytellers》— Marcos Mateu-Mestre** — 一本用黑白分镜讲构图的实战书。把所有东西画成剪影，只调整剪影的位置和大小来控制视觉焦点。这个方法和独立开发者在缩略图阶段做构图决策的方式完全一致。强烈推荐。★★★★★
2. **《Drawing Basics and Video Game Art》— Chris Solarski，第 3-4 章** — 少数直接讨论"游戏 × 古典构图"的书。Solarski 把古典绘画构图分析和现代游戏构图分析放在一起对比，展示了动态对称、三角形构图、引导线管理三个构图原则在游戏里的直接应用。★★★★★
3. **"The Rule of Thirds" — Cambridge in Colour**（https://www.cambridgeincolour.com/tutorials/rule-of-thirds.htm）— 一个专门讲三分法的摄影网站。不套玄学词汇，纯讲视觉逻辑。推荐读它的"A/B 对比"案例部分。★★★★☆
4. **Polycount Wiki: Composition**（https://wiki.polycount.com/wiki/Composition）— 游戏美术社区的构图条目——由从业者编写和迭代。包含构图 checklists，在过场动画和关卡设计中实用。★★★★☆
5. **"Composition for Game Artists" — YouTube 搜索 ModernDayJames** — 把古典绘画的构图结构提取出来叠在同一类型的游戏截图上的对比分析，非常直观。★★★★☆
