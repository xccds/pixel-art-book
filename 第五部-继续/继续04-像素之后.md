# 继续04 像素之后：向手绘与 3D 低模扩展

### 4.0 这一章解决什么问题

继续03 收尾时说了"Go make something"——你已经有了一套完整的像素艺术能力：观察、练手、风格、制作、继续。但像素不是终点。它是你"看 → 做 → 整 → 决策"这套能力的第一个载体。这一章很短，因为它只做一件事：**在你掌握像素之后，给你三条最自然的扩展方向——手绘数字绘画、3D 低多边形、矢量——以及每条的"何时转向"和"你的像素技能如何迁移过去"。**

这不是教你怎么画手绘、怎么建 3D 模型、怎么做矢量。这些门类在源书的第 17、20、22 章里有完整的工作流（本书已把它们砍出正文，降为本章的扩展阅读）。这一章给你的不是深度，是**方向感**：让你知道下一个工具该往哪边走，以及你手上这套像素功夫在新工具里哪些能用、哪些要重学。

> **程序员类比（三条路径各自的对应物）：** 手绘数字绘画 = 从强类型语言切到动态语言——约束少了，自由多了，但笔触和压力这些"内存"得你自己管；3D 低多边形 = 从 2D 数组升到 3D 数组——约束思维不变（用最少的单元表达），只是维度 +1；矢量 = 从位图（raster）切到矢量（vector）——分辨率无关，你画的是路径不是像素，像 SVG 之于 PNG。这三个类比贯穿本章，记住它们你就记住了每条路径的本质。

### 4.1 三条扩展路径——何时转向，什么会迁移

![能力扩展路径](images/继续/继续04-fig01-能力扩展路径.svg)

*图 继续04.1：能力扩展路径——像素（中心，已掌握）分叉到三条相邻方向，每条标注"何时转向"与"什么迁移"。约束训练是公共底座，三条路径共享它。*

下面逐条展开。每条只讲两件事：**什么时候你该切过去**，和**你练出来的像素功夫哪些能直接搬过去**。

#### 方向一：数字手绘（Krita）

**何时转向。** 你的游戏需要像素给不了的东西——柔和的边缘、连续的渐变、笔触的温度、painterly（油画感）的整体氛围。像素的硬边和有限调色板是它的风格语言，但当你的下一款游戏想表达"温暖的手""流动的水""湿润的泥土"时，硬边会变成障碍。这时候你把手绘数字绘画加进来。

> **程序员类比：手绘 = 从强类型语言切到动态语言。** 像素是强类型——每个像素位置、每个颜色索引都被网格严格约束，编译器（网格）替你挡掉了一切"模糊"。手绘是动态语言——笔触大小、压力、不透明度全是连续变量，自由度暴涨，但你得自己管"内存"：笔触怎么叠、压力怎么控制、边缘是硬还是软，没有网格替你兜底。约束少了，自由多了，**责任也多了**。

**像素技能如何迁移。** 好消息：你在练手部练出来的东西大部分直接能用。
- **明度（练手04）**：去色之后才见真章——这条规则在手绘里更狠，因为手绘的渐变更容易把明度搞糊。你在像素里学的"三区四模板"直接搬。
- **色彩（练手05）**：有限调色板的思维在手绘里反而是优势——大多数初学者手绘的问题是"颜色太多太花"，你练过"少即是多"，起手就比他们克制。
- **构图（练手07）**：视觉权重、引导线、三分法——这些和工具无关，是画面层的规则，100% 迁移。
- **形状心理学（练手02）**：圆=友善、方=稳重、三角=危险——这和你是用像素还是用笔刷画无关。

**什么要重学。** 只有一样：**笔触本身**。像素的"笔触"是单个像素的放置，手绘的笔触是压力、角度、流速的组合——这是新工具的"语法"，得从头练。但"语义"（明度、色彩、构图、形状）你已经会了。

**工具一句话。** Krita 是开源数字绘画的标准选择。对游戏美术你只需要关注四个功能：图层管理（分部件导出）、Wrap-around 无缝平铺（画纹理）、Export Slices（批量切 UI）、索引色模式（如果你要做像素/手绘混血）。其余几百个笔刷先放一边。源书 ch17 有完整的 Krita 游戏美术工作流，作为扩展阅读（见 4.6）。

#### 方向二：3D 低多边形（Blender）

**何时转向。** 你的游戏需要真正的 3D——可旋转的相机、可复用的多角度资产、物理交互、纵深透视。像素的 2D 网格在这些需求面前是维度的天花板。注意：如果你的需求只是"3D 当中间表示、最终产物还是 2D 像素 sprite"，那不是"转向"，那是**制作06 已经讲过的 3D→像素管线**——你不需要换方向，你只是在像素流水线上加了一个 3D 工具。这一节说的"转向"是你下一款游戏**主风格就是 3D 低模**，像素不再是最终产物。

> **程序员类比：3D 低模 = 从 2D 数组升到 3D 数组，但低模 = 用最少的体素（多边形）表达。** 约束思维不变——你还是在"用最少的单元做最准的表达"，只是维度从 (x,y) 变成了 (x,y,z)。低多边形的"面数少"和像素的"分辨率低"是同一种审美决策：不假装自己是高精度，诚实地把"有限单元"当成视觉语言。这就是为什么像素 artist 转 3D 低模比转写实 3D 容易——写实 3D 是"无限数组"的世界，约束思维在那里反而成了束缚。

**像素技能如何迁移。**
- **有限调色板（练手05）→ 顶点色克制。** 低模的顶点色（Vertex Color）路线就是"用最少的颜色指定每个面"——和你练过的有限调色板是同一个哲学，只是颜色施加在顶点上而不是像素上。源书 ch22 有"4KB 顶点色 vs 1.3MB 纹理"的 300 倍内存量化对比，是这个哲学的数字证明。
- **剪影（练手02）→ 3D 剪影可读性。** 剪影说了算——这条规则在 3D 里更严，因为玩家会从多个角度看你的模型。你在像素里练的"去色后认不认得出"的剪影测试，直接用于低模的轮廓设计。
- **约束藏瑕（继续03）→ 低模的"少面数是设计不是预算不够"。** 你在像素里学过"用 16×16 隐藏不会画解剖结构"——低模里同样的逻辑变成"用 300 面隐藏不会做高精度拓扑"。这不是偷懒，是主动的设计选择。源书 ch22 把这叫"几何自信"：你不为面数少道歉，因为面数少是你的声明。

**什么要重学。** 3D 独有的东西：**摄像机、光照在三维空间里的布置、UV 展开**（如果你用纹理路线）。这些是维度升级带来的新语法。但"语义"（形状、色彩、构图、剪影）你已经会了。

**工具一句话。** Blender 是开源 3D 的标准。低模入门你只需要四个操作：编辑模式（Tab）、挤出（E）、环切（Ctrl+R）、缩入（I）。不需要学雕刻、不需要学 UV（纯顶点色路线）、不需要学 PBR。源书 ch22 有完整的低模六条黄金法则和三条子路线（平面着色 / PBR / 手绘纹理），作为扩展阅读（见 4.6）。如果你走的是"3D 渲染成像素 sprite"的路线，回看制作06 的 Blender 3D→像素管线——那里已经把"3D 当源码、像素当目标码"的编译器类比讲透了。

#### 方向三：矢量（Inkscape）

**何时转向。** 你的游戏需要分辨率无关的资产——UI 图标、可缩放的 Logo、几何感强的图标系统、多分辨率适配的矢量图形。像素是位图（raster），放大就糊；矢量是路径方程，放大不糊。当你做一套要在 1080p 和 4K 上都清晰的 UI，或者一套要印到 T 恤上的 Logo，像素的网格反而成了累赘。

> **程序员类比：矢量 = 从位图（raster）切到矢量（vector）。** 像素是 PNG——固定分辨率，每个像素一个颜色值，放大就出现马赛克。矢量是 SVG——存的是路径方程和填充规则，渲染时实时计算，分辨率无关。你画的是"路径"不是"像素"——这就像从"手写数组"切到"写生成函数"：数组存的是结果，函数存的是规则，规则可以在任意分辨率下重新求值。

**像素技能如何迁移。**
- **形状心理学（练手02）**：矢量的核心是基本形状（圆、方、三角、路径）的组合——你在像素里练的形状心理学 100% 适用，而且矢量里你能更精确地控制形状（没有像素锯齿）。
- **硬边思维（练手08）**：像素艺术的硬边原则——边缘要干净、不要半透明亚像素——在矢量里是天然成立的（矢量路径本来就是硬边）。你练过的"边缘是视觉信息"的思维，在矢量里直接生效。
- **有限调色板（练手05）**：矢量 UI 同样受益于克制配色——一套图标用 2-3 个颜色比用 10 个颜色更专业。你的像素配色训练直接搬。

**什么要重学。** **路径编辑**。矢量的"笔"是贝塞尔曲线（Bézier curve）——控制点、手柄、曲率。这和像素的"逐像素放置"是完全不同的操作模型。但"画什么"（形状、构图、配色）你已经会了，重学的只是"怎么画"（路径操作）。

**工具一句话。** Inkscape 是开源矢量的标准选择。你只需要学：钢笔工具（画路径）、形状工具（调控制点）、布尔运算（合并/差集/交集形状）、导出 SVG/PNG。源书没有专门的矢量章节，但 Inkscape 官方教程和"Boolean Operations"是你起步的两个关键词。

### 4.2 如何选你的下一个方向——基于约束的迷你决策

三条路径不是"哪个更好"——是"你的下一款游戏需要什么"。用一个约束清单来选，和风格03 的风格选择同构：

| 你的下一款游戏需要…… | 转向 |
|----------------------|------|
| 柔和边缘、笔触温度、painterly 氛围 | → 手绘（Krita） |
| 可旋转相机、物理交互、多角度复用资产 | → 3D 低模（Blender） |
| 分辨率无关、UI 图标系统、几何 Logo | → 矢量（Inkscape） |
| 仍是 2D、仍是网格美学、只是想要更高分辨率 | → 留在像素，提高分辨率（风格02） |
| 3D 但最终产物是 2D sprite | → 留在像素，加制作06 的 3D→像素管线 |

**关键判断：你是"换工具"还是"换方向"。** 如果你的下一款游戏主风格还是像素，你只是想用 3D 省动画工时——那不是"转向 3D"，那是"在像素流水线上加工具"（制作06）。真正的"转向"是你的**主风格**变了：最终产物不再是像素 sprite，而是手绘插画 / 3D 模型 / 矢量图形。换工具是增量，换方向是迁移。先想清楚自己要的是哪个。

> **程序员类比：换工具 vs 换方向 = 加依赖 vs 换技术栈。** 给项目加一个 Redis 缓存是"换工具"——你的技术栈没变，只是多了一个组件。把后端从 Node 换到 Go 是"换方向"——你的核心语言变了，旧代码大部分要重写。同理，给像素游戏加 Blender 是"换工具"，把下一款游戏做成 3D 低模是"换方向"。前者低成本，后者高成本但高回报——别把两者搞混。

### 4.3 元点：约束训练的可迁移性

这一章最重要的不是三条路径的细节——是一个**元点**：

**你在像素里练的不是像素，是"在约束下做决策"。**

有限调色板、低分辨率、硬边、逐像素控制——这些约束逼你学会了在限制里做最优解。这个能力是通用的。手绘的约束是"笔触和压力你得自己管"，3D 低模的约束是"面数你得自己分配"，矢量的约束是"路径你得自己定义"。约束变了，**"在约束里做设计"的思维不变**。

这就是为什么像素是好的"第一语言"——它约束最硬，训练最狠。你从像素出发转任何一条路径，都会带着"我已经习惯在限制里思考"的优势。而那些从"无约束的自由手绘"起步的人，转到像素或低模时往往手足无措——因为他们没练过"约束下做决策"。

继续03 的"约束藏瑕"在这里升级：约束不只是藏缺陷的工具，约束是**所有视觉风格的生产方式**。每一种风格都是某种约束的组合——像素是分辨率约束，低模是面数约束，手绘是笔触约束，矢量是路径约束。**选风格就是选约束，选约束就是选你要练哪种决策。** 你在像素里练的那种，是最基础的一种——所以它迁移得最远。

### 4.4 练习

**L1：试一条路径，观察什么迁移（45 分钟）**

从三条路径里选一条你最感兴趣的，做一个最小的试验件，观察你的像素技能哪些直接能用、哪些要重学。

- **手绘方向**：Krita 新建 512×512 画布，用硬边圆笔刷画一个你在像素里画过的角色（比如制作02 的贯穿角色），但用压力笔刷画轮廓和阴影。观察：明度判断、配色、构图是不是直接生效？笔触控制是不是要重学？
- **3D 低模方向**：Blender 新建 Cube，用挤出 + 环切做一个你在像素里画过的道具（比如一个宝箱），给每个面指定顶点色。观察：剪影可读性、配色克制是不是直接生效？3D 空间操作是不是要重学？
- **矢量方向**：Inkscape 用钢笔工具画一个你在像素里画过的 UI 图标，导出 SVG。观察：形状心理学、硬边思维是不是直接生效？贝塞尔曲线控制是不是要重学？

**怎么检验。** 做完后写两句：①"从像素直接迁移过来的能力是___"②"必须从头学的新技能是___"。这两句话就是你下一步学习方向的起点——迁移过来的不用再练，从头学的才是你要投入时间的地方。

**最低完成线。** 一个能认出"这是我那个像素角色/道具/图标"的新工具试验件即过——不必好看，跑通"换工具、保语义、学新语法"的闭环就是目的。

### 4.5 小结

- **三条扩展路径，各有何时转向：** 手绘（需要柔和边缘/painterly）→ 3D 低模（需要真 3D 相机/物理/多角度资产）→ 矢量（需要分辨率无关/UI 系统/几何 Logo）。转向的信号是"像素给不了这个东西"，不是"像素不好"。
- **像素技能的迁移是语义层、不是语法层。** 明度、色彩、构图、形状心理学、剪影、有限调色板、约束藏瑕——这些是"画什么"的语义，跨工具通用。要重学的只是"怎么画"的语法：手绘的笔触、3D 的空间操作、矢量的贝塞尔曲线。
- **三条路径的程序员类比：** 手绘 = 强类型→动态语言（约束少自由多，自己管内存）；3D 低模 = 2D 数组→3D 数组（约束思维不变，维度+1）；矢量 = raster→vector（存路径不存像素，分辨率无关）。
- **换工具 ≠ 换方向。** 给像素游戏加 3D 工具是换工具（增量），把下一款游戏主风格改成 3D 是换方向（迁移）。前者低成本，后者高回报但高成本——先想清楚你要的是哪个。
- **元点：你在像素里练的是"在约束下做决策"，这是通用艺术技能。** 像素是约束最硬的第一语言，所以它的训练迁移得最远。

> **如果只记住一句话：** 像素是你的第一语言，但不是唯一语言——你练出来的不是像素本身，是"在约束下做决策"的元技能，它在你转向手绘、3D 低模、矢量任何一条路径时都跟着你。

> **上手行动：** 今晚做 L1——45 分钟选一条路径做一个最小试验件，写下"什么迁移了、什么要重学"两句话。这两句话就是你的下一份学习清单的起点。

### 4.6 扩展阅读

1. **源书 ch17《数字绘画与 3D 入门工作流》** —— Krita 游戏美术模式（图层/无缝平铺/切片导出/索引色四功能）+ Blender 低模五步法 + 2D/3D 混合管线（Blender 打草稿、Krita 精修）+ PBR 极简入门。**为什么推荐：** 本章 4.1 方向一和方向二只给了方向感，ch17 给你的是手绘和 3D 的具体工作流——当你决定切过去时，从那里起步。
2. **源书 ch20《3D 游戏美术风格全景》** —— 3D 六大风格（低多边形/体素/风格化/赛璐珞/手绘纹理/半写实/写实）的成本与独立性适配矩阵。**为什么推荐：** 本章只讲了低多边形这一条 3D 路径（因为它对像素 artist 最友好），ch20 让你看到 3D 的完整光谱——如果你在低模之外想了解体素或风格化 3D，从那里开始。
3. **源书 ch22《低多边形风格深度指南》** —— 低模三条子路线（平面着色/PBR/手绘纹理）+ 六条黄金法则 + 顶点色 vs 纹理的 300 倍内存量化对比 + Monument Valley / A Short Hike / 往室三案例。**为什么推荐：** 本章 4.1 方向二只给了低模的"何时转向+迁移"，ch22 给你的是低模的"怎么做才不丑"——低面数的每个面都是视觉声明，这是从像素"每个像素都有意放置"直接对应的审美哲学。
4. **Inkscape 官方教程（Basic Tutorial + Boolean Operations）** —— 矢量入门。**为什么推荐：** 源书没有专门矢量章节，Inkscape 自带教程是起步标准——Basic Tutorial 覆盖路径和形状，Boolean Operations 覆盖合并/差集/交集（这是矢量"组合形状"的核心操作，类比像素的"组合像素"）。
5. **制作06《非手绘管线：3D、AI 与程序化》** —— Blender 3D→像素管线。**为什么推荐：** 如果你不是"转向 3D"而是"在像素流水线上加 3D 工具"，制作06 已经把"3D 当源码、像素当目标码"的编译器类比和 Compositor 节点链讲透了——回看那一章比从 3D 低模起步更对口。
