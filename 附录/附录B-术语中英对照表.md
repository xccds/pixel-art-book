# 附录B 术语中英对照表

> 按类别整理，不是按字母。每个术语用一句话定义——详细讲解在正文各章中。_斜体_ 表示本书建立或强调的核心概念。

---

## B.1 八概念——看的基础词汇（练手02-09，风格01-04）

| English | 中文 | 一句话定义 |
|---------|------|-----------|
| Line | 线条 | 点的运动轨迹；在像素艺术中，线条质量由斜率比率和 AA 决定。 |
| Shape | 形状 | 二维封闭轮廓——圆（友好）、方（稳定）、三角（攻击性）的心理暗示。 |
| Space | 空间 | 正形与负形的关系，也包括前景/中景/远景的纵深分层。 |
| Positive Space | 正形 | 画面中有物体的区域——主体、角色、建筑等占据的空间。 |
| Negative Space | 负空间 | 画面中没有物体的区域——不是"空白"，是呼吸、情境和视线休息区。 |
| Value | 明度 | 颜色的亮暗程度；去掉色相和饱和度后仍能读出焦点才算合格。 |
| Color | 色彩 | 色相、饱和度、明度三者的组合系统，用于传达情绪和功能提示。 |
| Texture | 质感 | 表面的视觉触觉暗示——光滑/粗糙/透明/金属/毛茸茸。 |
| Composition | 构图 | 元素在画面中的组织策略——焦点放哪、视线怎样流动、空间如何分配。 |
| Style | 风格 | 以上七概念在特定方向和约束下的整体视觉语言——一个"打包好的视觉决定集"。 |
| Visual Weight | 视觉权重 | 画面中不同元素"抢夺注意力"强度的对比——玩家先看哪个、再看哪个、忽略哪个。 |
| Form | 体积 | 在二维平面上表现三维体量感——Shape 是轮廓，Form 是"这个物体有重量"。 |

## B.2 色彩相关（练手04-05，风格01-02）

| English | 中文 | 一句话定义 |
|---------|------|-----------|
| Hue | 色相 | 色彩的基本属性——红、蓝、绿等纯色的"身份"。 |
| Saturation | 饱和度 | 颜色的纯净程度；高饱和鲜艳，低饱和接近灰色。 |
| Value / Brightness | 明度 | 颜色的亮暗程度。 |
| HSV | 色相-饱和度-明度模型 | 以色相为环、饱和度和明度为轴的色彩空间，比 RGB 更接近人直觉。 |
| HSL | 色相-饱和度-亮度模型 | 与 HSV 类似但亮度定义不同——HSL 的 L=50 是纯色，HSV 的 V=100 是纯色。 |
| Gamut | 色域 | 一个色板能覆盖的颜色范围——你的风格文档限定了"这台显示器能显示的所有颜色中的哪一小块"。 |
| Hue Shifting | 色相偏移 | 在阴影和高光中改变色相而非只改变明度的技术——暖色高光偏黄、冷色阴影偏蓝是经典的"东式 hue-shift"。 |
| Cold / Warm Colors | 冷暖色 | 冷色（蓝绿紫）后退，暖色（红橙黄）前进——这是空间感知的本质机制。 |
| Analogous Colors | 类比色 | 色环上相邻的色彩组合，天然和谐但不适合做强对比。 |
| Complementary Colors | 补色 | 色环上正对的色彩组合，对比最强——用得好制造张力，用不好刺眼。 |
| Triadic Colors | 三色组 | 色环上均匀分布的三色组合，均衡但需选定主色控制视觉重心。 |
| Shadow Color | 暗部色 | 一个物体的阴影区域应该用的色——通常不是"加黑"，而是朝补色方向偏移。 |
| Color Ramp | 色板阶梯 / 渐变 | 一组按明度排列的颜色过渡，是色板的核心结构——你要的不是"16 个好看的颜色"，是"4 组 × 4 阶 ramp"。 |

## B.3 构图相关（练手01，练手03，练手07）

| English | 中文 | 一句话定义 |
|---------|------|-----------|
| Focal Point | 焦点 | 画面中"第一眼看过去"的位置——用明度/色彩/细节密度等手段制造。 |
| Rule of Thirds | 三分法 | 把画面纵横各分三等分，交点处是天然的视觉重心位。 |
| Leading Lines | 引导线 | 画面中隐含的线条方向，引导视线从 A 走到 B。 |
| Framing | 框景 | 用画面元素（门框、树枝、窗）把视线框在主体上。 |
| Visual Hierarchy | 视觉层级 | 元素按重要性在视觉上的排序——最重要的最大最亮最靠前。 |
| Silhouette | 剪影 | 用单一颜色填充的实体轮廓——角色设计的第一验证：只看剪影能认出它是谁吗？ |

## B.4 像素特有（练手01，练手06，练手08，制作01-03）

| English | 中文 | 一句话定义 |
|---------|------|-----------|
| Pixel | 像素 | 数字图像中最小的颜色单元——像素艺术的一切从"一个格子一种颜色"开始。 |
| Sprite | 精灵 | 2D 游戏中一个带有动画帧信息的角色/物体的像素表示。 |
| Sprite Sheet | 精灵表 | 把多帧动画或多种状态的精灵集中在一张大图上，运行时切帧。 |
| Tileset | 瓷砖集 | 一套可重复拼接的小图块，用于构建 2D 游戏的场景地图。 |
| Tile | 瓷砖 / 瓦片 | Tileset 中的单个可重复图块。 |
| Palette | 色板 | 一个精灵/场景/游戏允许使用的完整颜色集合，像素艺术的色板通常 ≤ 32 色。 |
| Ramp | 色阶 / 渐变阶梯 | 同一色相从亮到暗的一组过渡色——色板的基本组成单元。 |
| Anti-Aliasing (AA) | 抗锯齿 | 在锯齿处添加过渡色像素使线条平滑的技术——像素艺术中"可控的模糊"。 |
| Banding | 条带 / 并排粘连 | 两排像素平行紧贴形成的不自然条带状视觉噪音——AA 的副作用，清理目标之一。 |
| Dithering | 抖动 | 用棋盘格/交错等像素图案混合两种颜色，创造中间色和渐变的错觉。 |
| Jaggies | 锯齿感 | 斜线在低分辨率下形成的阶梯状视觉噪音——像素艺术的头号敌人。 |
| Mixels | 混合像素比例 | 同一作品中混合不同像素比例的问题（如 1x 像素旁边放 2x 像素）——破坏"所有像素是同一世界的原子"这个基础假设。 |
| Onion Skinning | 洋葱皮 | 动画软件中同时显示前一帧和后一帧半透明影像的功能，用于逐帧动画的定位参考。 |
| Sub-Pixel | 子像素动画 | 在动画中移动少于 1 像素的技术——通过 AA 渐变暗示"这个边缘在两个像素之间"。 |
| Hue Shifting | 色相偏移 | （见色彩相关表）像素艺术中"加黑 = 低级、朝冷色偏移 = 高级"的核心色彩技法。 |
| Isometric | 等距视角 | 展示物体三个面的伪 3D 透视方法——像素艺术中最经典的"2.5D"方案。 |
| Dimetric | 两等距视角 | 与等距类似但只有两个轴的缩放比例相同——2:1 的 dimetric 是像素 RPG 的事实标准。 |
| Nearest Neighbor | 最近邻采样 | 缩放图像时不做任何平滑过渡——像素艺术的唯一正确采样方式。 |

## B.5 3D 相关（制作06 非手绘管线）

| English | 中文 | 一句话定义 |
|---------|------|-----------|
| Flat Shading | 平面着色 | 每个面只使用单一颜色（不插值），是低多边形风格的核心渲染技术。 |
| Low Poly | 低多边形 | 以极少面数（通常几百到几千面）构建的 3D 模型，牺牲细节换取清晰轮廓。 |
| PBR | 基于物理的渲染 | 一套模拟真实光学材质属性的工作流，含反照率/金属度/粗糙度等贴图通道。 |
| Vertex Color | 顶点色 | 直接往模型顶点上着色的技术，省去贴图——低多边形风格的常见选择。 |
| UV Unwrapping | 展 UV | 把 3D 模型的表面"摊平"到二维纹理坐标上——类似剥橙子皮。 |
| Normal Map | 法线贴图 | 用 RGB 值编码表面凹凸方向的贴图，让低面数模型看起来有高面数细节。 |
| Ambient Occlusion (AO) | 环境光遮蔽 | 模拟角落和缝隙处环境光被阻挡的效果——让物体看起来"落地了"。 |

## B.7 动画相关（制作05）

| English | 中文 | 一句话定义 |
|---------|------|-----------|
| Keyframe | 关键帧 | 动画中定义动作"起"和"落"的核心帧——关键帧对了，中间帧再粗也不太致命。 |
| In-between | 中间帧 | 两个关键帧之间的过渡帧，让动作看起来平滑——逐帧动画中最耗时的环节。 |
| Frame-by-Frame | 逐帧动画 | 每一帧都是手绘/手调的独立画面，像素动画的主要工作方式。 |
| FPS | 帧速率 | 每秒播放的帧数——像素游戏通常 8-12 FPS，不是"越高越好"。 |
| Squash & Stretch | 挤压拉伸 | 物体在运动中通过形变传达速度和重量感——球落地变扁、弹起拉长。 |
| Anticipation | 预备动作 | 在主要动作发生前的反向运动——跳之前下蹲，挥拳之前手臂后缩。 |
| Follow-Through | 跟随动作 | 主要动作结束后的惯性延续——头发/衣摆/尾巴的"滞后感"。 |
| Easing | 缓入缓出 | 动画的开始和结束阶段速度渐变——自然界没有瞬时启动和瞬时停止。 |
| Walk Cycle | 走路循环 | 一组可无限循环的走路帧——8 帧走路循环是最常见的像素游戏角色待机/移动方案。 |
| Skeletal Animation | 骨骼动画 | 用虚拟骨架驱动模型变形的动画方式——适合 3D 和 Spine 2D，不是像素逐帧。 |

## B.8 游戏美术相关（制作04，制作09，继续01-03）

| English | 中文 | 一句话定义 |
|---------|------|-----------|
| HUD | 抬头显示 | 屏幕表面的非游戏世界 UI——血量条、小地图、技能图标。 |
| Diegetic UI | 叙境内 UI | 存在于游戏世界内部的界面——角色手腕上的手表面板，而非屏幕角落的图标。 |
| Non-diegetic UI | 叙境外 UI | 只存在于屏幕表面、角色"看不见"的界面——就是 HUD。 |
| Affordance | 功能暗示 | 一个物体的外观暗示着它的用途——红色会爆炸、门形的东西能穿过。 |
| Parallax Scrolling | 视差滚动 | 不同深度的图层以不同速度滚动，模拟三维空间——远处慢、近处快。 |
| Readability | 可读性 | 观众理解像素画内容的难易程度——32×32 里画"一把剑"，3 秒内能被认出来才算及格。 |
| _Gold Standard_ | _黄金标准_ | _本书概念：最能代表你游戏"日常画面"的一张截图——所有新资产都和它比。_ |
| Game Jam | 游戏极限开发 | 在极短时间内（通常 48-72 小时）完成一个微型游戏的完整开发——本书推荐的新手起步方式。 |
